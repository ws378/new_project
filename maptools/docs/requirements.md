# ROS2 地图编辑工具需求文档

## 项目概述

一个用于编辑ROS2建图生成地图的轻量级工具，支持标注禁行区、基站位置、虚拟墙等，并可修正地图坐标系。

## 技术栈

| 组件 | 选择 | 理由 |
|------|------|------|
| GUI | Tkinter | Python自带，无Qt冲突，分发简单 |
| 图像处理 | Pillow + OpenCV | 图像读写、变换 |
| 向量数据 | JSON | 可读性好，便于ROS2节点解析 |
| Python环境 | conda: maptools | - |

## 输入格式

标准ROS2地图格式：

```yaml
# map.yaml
image: map.pgm
mode: trinary
resolution: 0.05          # 米/像素
origin: [-15.7, -16.5, 0] # 地图原点在世界坐标系中的位置 [x, y, yaw]
negate: 0
occupied_thresh: 0.65
free_thresh: 0.25
```

- **PGM图像**: 灰度图，白色(255)=可通行，黑色(0)=障碍物，灰色(205)=未知

## 功能列表

### 1. 基础功能

| 功能 | 描述 |
|------|------|
| 地图加载 | 加载PGM + YAML文件 |
| 缩放 | 鼠标滚轮缩放，支持适应窗口 |
| 平移 | 鼠标拖拽平移地图 |
| 坐标显示 | 鼠标位置实时显示世界坐标(米) |

### 2. 可视化辅助

| 功能 | 描述 |
|------|------|
| 网格叠加 | 显示1米网格线 |
| 比例尺 | 显示当前比例尺 |
| 原点显示 | 可视化坐标系原点和XY轴方向 |

### 3. 地图编辑（栅格层）

| 功能 | 描述 | 快捷键建议 |
|------|------|-----------|
| 画笔 | 绘制障碍物（黑色）或可通行区（白色） | B |
| 橡皮擦 | 清除为可通行区 | E |
| 笔刷大小 | 可调节笔刷半径 | [ ] |

### 4. 坐标系调整

| 功能 | 描述 |
|------|------|
| 原点平移 | 拖拽设置新原点位置 |
| 地图旋转 | 旋转地图使墙壁对齐水平/垂直 |
| 参考线 | 画参考线辅助计算旋转角度 |
| 吸附 | 旋转时支持15°增量吸附 |

> 注意：坐标系调整后，所有已有标注需同步变换

### 5. 向量标注

| 标注类型 | 描述 | 数据 |
|----------|------|------|
| 禁行区 | 多边形区域，机器人不可进入 | 顶点坐标列表 |
| 虚拟墙 | 线段障碍 | 起点、终点坐标 |
| 基站 | 充电桩位置 | 坐标 + 朝向角度 |
| 仅通行区 | 多边形区域，机器人可通行但不执行清洁作业（如地贴区域） | 顶点坐标列表 |

### 6. 辅助工具

| 功能 | 描述 |
|------|------|
| 测距 | 测量两点间实际距离(米) |
| 撤销 | 撤销上一步操作 (Ctrl+Z) |
| 重做 | 重做已撤销操作 (Ctrl+Y) |

### 7. 数据管理

| 功能 | 描述 |
|------|------|
| 保存项目 | 保存当前编辑状态 |
| 加载项目 | 加载已保存项目继续编辑 |
| 导出 | 导出主地图供 nav2 静态地图使用；额外PGM图层仅供内部/后续插件扩展 |

## 坐标变换权威与约束

- 权威变换源仅为 `map.yaml`（含 `origin` 与分辨率），保存项目时所有标注坐标都已应用该变换。
- `annotations.json.coordinate_transform` 仅作为历史/调试信息，不参与显示与导出，保存时保持为 `0` 偏移与 `0` 旋转。
- 旋转采用“烘焙到图像”的方式：图像像素旋转后同步更新 `map.yaml` 与标注坐标，避免双重变换。
- MVP 约束：`map.yaml` 的 `origin[2]` 以 `0` 为基准；若加载时非零，需要在加载阶段先做归一化（烘焙旋转并将 `origin[2]` 置零）。

## 坐标转换

像素坐标与世界坐标转换：

```python
# 像素 -> 世界坐标
world_x = origin[0] + pixel_x * resolution
world_y = origin[1] + (image_height - pixel_y) * resolution  # Y轴翻转

# 世界坐标 -> 像素
pixel_x = (world_x - origin[0]) / resolution
pixel_y = image_height - (world_y - origin[1]) / resolution
```

## 版本更新记录

### v1.1 新增需求

#### 需求 1：仅通行区图层（PassOnlyZone）

**背景**：
在实际场景中，存在一些区域机器人可以通过但不应执行清洁作业，例如地贴区域、临时展示区等。需要新增一个图层来标注这类区域。

**功能描述**：
| 项目 | 说明 |
|------|------|
| 图层名称 | 仅通行区 / PassOnlyZone |
| 几何类型 | 多边形（与禁行区类似） |
| 语义 | 机器人可通行，但不执行清洁作业 |
| 显示颜色 | 建议使用黄色半透明填充，与禁行区（红色）区分 |
| 导出格式 | 独立的 `map_pass_only.pgm` 栅格图层 |

**用户交互**：
- 工具栏新增「仅通行区」按钮（快捷键建议：O）
- 绘制方式与禁行区一致（多边形绘制）
- 图层面板中可独立控制可见性

**数据结构**：
```json
{
  "pass_only_zones": [
    {
      "id": "pass_only_1",
      "name": "地贴区域A",
      "polygon": [[2.0, 3.0], [4.0, 3.0], [4.0, 5.0], [2.0, 5.0]]
    }
  ]
}
```

#### 需求 2：项目二次编辑支持

**背景**：
当前系统存在以下问题：
1. 栅格编辑层（画笔/橡皮擦修改）在导出时与原图合并，无法区分用户修改和原始障碍物
2. 导出的 PGM 文件是最终渲染结果，丢失了源数据结构
3. 用户无法对已保存/导出的项目进行二次编辑

**需求目标**：
实现完整的项目保存/加载机制，支持：
- 保存完整的编辑状态（原始地图 + 编辑层 + 标注）
- 加载已保存项目并继续编辑
- 区分「保存项目」与「导出部署文件」两个操作

**项目文件结构**：
```
project_name/
├── project.json          # 项目元数据
├── <map_name>.pgm        # 项目内地图图像
├── <map_name>.yaml       # 项目内地图元数据
├── edit_layer.npy        # 栅格编辑层（NumPy 格式）
├── annotations.json      # 矢量标注
└── export/               # 导出的部署文件
    ├── map.pgm
    ├── map.yaml
    ├── map_forbidden.pgm
    ├── map_virtual_wall.pgm
    └── map_pass_only.pgm
```

**project.json 结构**：
```json
{
  "name": "project_name",
  "map_yaml_path": "<map_name>.yaml",
  "version": "1.0"
}
```

**用户操作流程**：

| 操作 | 菜单项 | 快捷键 | 说明 |
|------|--------|--------|------|
| 新建项目 | 文件 → 新建 | Ctrl+N | 从 PGM/YAML 创建新项目 |
| 打开项目 | 文件 → 打开项目 | Ctrl+O | 打开已保存的项目目录 |
| 保存项目 | 文件 → 保存项目 | Ctrl+S | 保存当前编辑状态 |
| 另存为 | 文件 → 另存为 | Ctrl+Shift+S | 保存到新目录 |
| 导出部署 | 文件 → 导出 | Ctrl+E | 生成用于机器人部署的文件 |

**edit_layer.npy 格式说明**：
- 使用 NumPy 二进制格式保存，高效且精确
- 像素值含义：
  - `255` = 未修改（透明，显示原图）
  - `0` = 用户绘制为障碍物（黑色）
  - `254` = 用户绘制为可通行（白色）
