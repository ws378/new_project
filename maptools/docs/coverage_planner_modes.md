# 覆盖路径算法模式说明

本地图编辑器当前支持以下覆盖路径算法模式：

- `auto`
- `basic`
- `shelf_aware`
- `shelf_aware_turn_cost`
- `channel_topology_graph`

## 1. basic

适用场景：

- 普通房间
- 开阔区域
- 通道结构不明显的区域
- 需要快速生成一条基础覆盖路径的场景

特点：

- 逻辑简单
- 参数少
- 对货架、条带式通道的结构感知较弱

## 2. shelf_aware

适用场景：

- 超市货架区
- 仓储货架区
- 存在明显窄通道和长条带结构的区域
- 希望减少横切、长跳和局部异常折返的场景

特点：

- 使用局部方向图感知通道主方向
- 对长跳和横切增加惩罚
- 对历史轨迹邻近和局部坏形状做约束
- CTG auxiliary 失败时记录 warning 后继续运行
- 质量守卫只读诊断，不改路径
- 允许通过少量关键参数做场景调节

## 3. shelf_aware_turn_cost

适用场景：

- 希望保留 ShelfAware 原路径组织和连接经验；
- 同时希望覆盖节点更接近 turn-cost 研究中验证过的规则网格；
- 需要显式尝试 `ShelfAware+TurnCost`，而不是让 `auto` 自动选择。

特点：

- 不使用完整 official turn-cost planner。
- 不接入研究后处理，例如 batch reconnect、节点吸附、删点或 lane 平移。
- 只把 ShelfAware 覆盖节点生成强制切换为 `turn_cost_repaired_grid`。
- 固定启用 CTG territory/junction auxiliary。
- 固定关闭孤立跳变点治理，避免和 turn-cost 节点替换实验口径混淆。

## 4. 编辑器中暴露的 ShelfAware 关键参数

编辑器当前仅暴露少量关键参数，避免把内部所有策略细节都放到 UI 中：

- `启用局部方向图`
  - 是否启用局部方向场引导路径。
  - 对货架区通常建议开启。

- `局部方向权重`
  - 越大越倾向顺着局部通道方向前进。
  - 过大可能让路径过于保守。

- `长跳惩罚权重`
  - 越大越不容易发生跨条带远跳。
  - 对减少后期突然跳转比较有效。

- `横切惩罚权重`
  - 越大越不容易横切通道。
  - 适合货架通道场景。

- `历史轨迹净空权重`
  - 越大越不希望跳转落点贴近旧轨迹。
  - 有助于减少接近交叉或贴边回补。

- `长跳切段阈值倍率`
  - 用于识别哪些跨段连接应被视为跳转段。
  - 越小越容易把长连接识别为跳转。

- `启用重复回接`
  - 是否允许有限重复访问作为桥接手段。
  - 在部分货架区有帮助，但不是所有区域都需要开启。

## 5. 选型建议

可以按下面的经验优先选择：

- 房间、办公区、普通大开口区域：优先 `basic`
- 超市货架区、规则条带通道区域：优先 `shelf_aware`
- 需要尝试当前 turn-cost 节点生成研究结果：显式选择 `shelf_aware_turn_cost`

如果在货架区出现以下问题，可以优先尝试 `shelf_aware`：

- 横切太多
- 后期长跳明显
- 局部区域没有扫净就远跳回补
- 路径贴近旧轨迹，视觉上疑似交叉
