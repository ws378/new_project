# 交汇重建模块方案

目标：从基础几何世界出发，先建立初始 `node_info` 与 `edge_info`，再完成节点聚类/节点合并、交汇中心求解、支撑结构求解、polygon 写回，以及边路径裁剪与边几何重建。  
本方案不负责正式 `graph_info` 组装，只负责把第 2 步需要稳定下来的节点与边写清楚。

## 1. 模块边界

**负责**
- 基于骨架世界建立初始 `node_info / edge_info`
- 在初始节点基础上执行交汇聚类与节点合并
- 在节点合并过程中处理节点失效、边重定向和边失效
- 对保留交汇节点按研究主线求：
  - 截断点
  - 扇区圆心/节点中心
  - 支撑结构
  - 支撑点
  - polygon
- 对 `dead_end` 节点建立最小节点占据区 polygon
- 用 `split_path_by_hits(...)` 按研究主线重建有效边几何
- 对边切分后暴露出的交汇内部短边执行二次节点合并
- 回写有效节点-边关联关系
- 在正式输出前 compact，只保留有效 `node_info / edge_info`

**不负责**
- 正式 `graph_info` 组装
- `directed_lane`
- `coverage_route`

## 2. 输入与输出

**输入（对外）**
- `geometry_preparation_result`

**输出（正式对象）**
- `junction_rebuild_result`

正式字段：
- `node_info_list` - 第 2 步输出的有效节点对象列表。
- `edge_info_list` - 第 2 步输出的有效边对象列表。
- `debug_info` - 交汇合并、节点几何、边切分、二次合并和 compact 的过程证据。
- `validation_info` - 第 2 步正式对象闭环校验结果。
- `meta` - 阶段名称、上下文键等元信息。

## 3. 处理流程（高层）

1. 基于骨架世界建立初始 `node_info / edge_info`
2. 在初始节点上执行交汇聚类与节点合并
3. 对每个保留交汇节点：
   - 根据是否为合并节点，确定旧中心集合
   - 求截断点
   - 求扇区/支撑结构
   - 求 polygon
   - 必要时进行第二轮中心更新
4. 对 `dead_end` 节点建立最小节点占据区 polygon
5. 用 `split_path_by_hits(...)` 重建有效边几何
6. 基于切分后的 `outer_path_rc` 检查交汇内部短边，必要时合并其两端交汇节点并重建几何
7. 回写有效节点-边关系，并 compact 输出 `node_info_list / edge_info_list`

## 4. 交汇重建规则

### 4.1 先建立初始 node/edge，再做节点合并
- 交汇重建的第一步必须先得到初始 `node_info / edge_info`
- 节点聚类与节点合并发生在初始对象建立之后
- 截断点、扇区、支撑结构、polygon 都只是更新节点和边的过程量
- 第 2 步结束时，对外正式对象仍然只有：
  - `node_info`
  - `edge_info`

### 4.2 节点合并必须显式处理节点与边的失效
- 节点合并不只是更新保留节点的位置
- 被吞并的节点必须失效
- 与被吞并节点相关的边必须同时处理：
  - 组内边失效
  - 外连边重定向到保留节点
  - 重定向后形成的自环边或重复边失效
- 工程上允许内部先用 `active/inactive` 处理，再在正式输出前 compact

### 4.3 合并节点不会插入独立的“局部骨架重建阶段”
- 节点合并之后，不单独插入一层 `rebuild_merged_junction_skeleton`
- 研究算法本意是：
  - 先完成节点合并
  - 再在保留节点上直接进入截断点/支撑结构/polygon 主链
- 是否为合并节点，只影响：
  - 旧中心集合怎么取
  - 截断点怎么找
  - 节点中心怎么求
- 不能把这件事扩展成一套新的中间几何阶段

### 4.4 polygon 是正式节点约束
- `node_info.polygon_vertices_rc` 是正式字段，不是调试字段
- 对 `junction` 节点，它来自：
  - 截断点
  - 扇区/支撑结构
  - polygon 求解
- 对 `dead_end` 节点，它表示最小节点占据区
- 后续它继续用于约束边路径、边有效区域和覆盖规划扩展

### 4.5 边几何必须拆成三段表达
- `edge_info.inner_path_rc`
  - 位于节点 polygon 内部的连接段
- `edge_info.outer_path_rc`
  - 位于节点 polygon 外部的主体几何路径
- `edge_info.path_rc`
  - 面向后续模块使用的完整边路径

### 4.6 边切分必须按 hits / fallback bridge 语义显式表达
- 若 `src/dst` 两侧都找到有效交点，则按交点正常切分
- 若某一侧没有交点，则按研究主线保留 `fallback bridge`
- `fallback bridge` 的目的不是更优几何，而是保证图结构不断
- 不应引入新的 polygon fallback 概念来替代这条主线

### 4.7 长度门限必须使用运行尺度
- 第 2 步内部所有长度类门限都应优先使用米制配置
- 像素级判断只能作为局部几何实现细节，不能作为对外配置主口径
- 历史 `8x` 经验值只允许作为迁移说明，不能继续作为新规则的解释来源
- 若需要从历史像素参数迁移，必须先换算到运行尺度，再用 `resolution_m_per_px` 对齐到米制语义
- 角度类门限保持角度语义，不做缩放

### 4.8 边切分后的交汇内部短边必须回到节点合并处理
- `outer_path_rc` 过短不是 coverage 阶段的过滤理由
- 如果一条 `junction -> junction` 边在 polygon 切分后只剩极短 `outer_path_rc`，说明两端交汇 polygon 已经几乎相接
- 这类边更像同一个交汇结构内部的骨架连接，而不是两个交汇之间的可覆盖通道
- 处理位置必须在 `junction_rebuild`，不能留到 `coverage_planning` 再跳过 sweep
- 二次合并的基本前置条件是：
  - `edge` 是 active edge
  - `src_node / dst_node` 都是 active junction
  - `outer_path_length_m < post_geometry_merge_outer_path_min_length_m`
  - `split_used_fallback = false`
  - `src_hit_index / dst_hit_index` 都存在，且 `src_hit_index < dst_hit_index`
  - `outer_path_rc` 必须由两端可靠 split hit 夹出，而不是 fallback bridge 或异常顺序裁出来
- 默认建议：
  - `post_geometry_merge_outer_path_min_length_m = 0.5`
  - `post_geometry_merge_full_path_max_merge_length_m = 3.0`
  - `post_geometry_merge_outer_path_anomaly_ratio = 0.1`
  - `max_post_geometry_merge_iterations = 2`
- 需要区分两类命中：
  - A 类正常二次合并：`outer_path_length_m < 0.5m` 且 `full_path_length_m <= 3.0m`
  - B 类异常 polygon/split 侵蚀：`outer_path_length_m < 0.5m` 但 `full_path_length_m > 3.0m`，或 `outer_path_length_m / full_path_length_m < 0.1`
- A 类应直接触发二次节点合并
- B 类不应直接普通保留，也不应套用 A 类规则盲目合并
- B 类应写入 `debug_info.post_geometry_merge.iterations[].anomaly_findings`
- B 类默认以 validation warning 形式暴露；若启用 `post_geometry_merge_strict_anomaly_fail`，应直接失败，避免基线治理阶段静默放过异常
- 正确动作是：
  - 将该边判定为 `internal_after_post_geometry_merge`
  - 合并它的 `src_node_id / dst_node_id` 两端 `junction` 节点
  - 将该短边置为 inactive 内部边
  - 将两端其它外连边重定向到 survivor 节点
  - 清理重定向后产生的自环边和重复边
  - 重跑保留节点 polygon 与相关边的 `inner_path_rc / outer_path_rc / path_rc`
- 这不是“删边导致断图”，而是“合并节点后内部边失效”
- 拓扑语义应从：
  - `A -- node6 -- short_edge -- node11 -- B`
  - 收口为 `A -- merged(node6,node11) -- B`
- 该规则应作为二次合并 pass，发生在第一轮 `rebuild_edge_paths(...)` 之后
- 二次合并应允许受限迭代：
  - 每一轮合并后必须重跑 polygon 与 edge split
  - 默认迭代上限固定为 `max_post_geometry_merge_iterations = 2`
  - 只有在专门基线验证确认需要时，才允许通过配置显式扩大
  - 若达到上限后仍有 A 类候选，应写入 debug warning，不能静默继续
- 二次合并不能简单替代第一轮 geodesic merge：
  - 第一轮 merge 使用骨架 geodesic 距离，发生在 polygon 求解之前
  - 二次 merge 使用 edge split 后的 `outer_path_rc` 证据，专门处理 polygon 几乎相接的短内部边

## 5. 关键派生与约束

### 5.1 节点合并是第 2 步的第一类核心操作
- 初始对象建立后，先做节点聚类与节点合并
- 节点中心调整、polygon 和边裁剪都发生在此之后
- 第 2 步所有后续几何更新，都只作用在保留节点和有效边上

### 5.2 节点区和 polygon 是两层边约束
- 初始节点区先用于初始建边：从骨架中扣掉节点区，切出残余骨架链
- 节点 polygon 再用于最终几何更新：把有效边进一步拆成内部段和外部段
- `dead_end` 虽然不走完整交汇主链，但仍需要一个最小节点占据区 polygon

### 5.3 节点与边的主键在第 2 步已经稳定
- `node_id` 在第 2 步建立后，下游只继承不重编号
- `edge_id` 在第 2 步建立后，下游只继承不重编号
- survivor 默认选择 `node_id` 最小的节点
- survivor 的选择只负责稳定身份，不代表合并后的几何真值
- 合并后的几何中心必须由合并成员旧中心集合重新求解或更新
- 若后续发现需要更复杂身份策略，再把 survivor 选择独立扩展

## 6. 失败条件（报错）

- 交汇合并后没有任何交汇节点
- 第 2 步最终骨架为空
- 无法建立任何 `node_info`
- 无法建立任何 `edge_info`
- `junction` 节点无法得到有效截断点
- `junction` 节点 polygon 为空、自交或无法约束对应边
- 边裁剪后 `path_rc` 长度小于 `2`
- 节点 `degree` 与 `incident_edge_ids` 数量不一致

## 7. 伪代码与数据结构

### 7.1 核心思路

第 2 步不是“先分析 cluster，再顺便产出 node/edge”。  
它的真实流程应按正式对象顺序理解：

1. 先建立初始 `node_info / edge_info`
2. 再在初始节点上执行节点聚类与节点合并
3. 再对保留交汇节点执行研究主线：
   - 截断点
   - 扇区/节点中心
   - 支撑结构
   - polygon
   - 必要时第二轮
4. 再对 `dead_end` 节点建立最小节点占据区 polygon
5. 按 `hits / fallback bridge` 主线重建边路径
6. 基于 edge split 后的 `outer_path_rc` 与可靠 split hit 证据执行 post-geometry merge
7. 每轮 post-geometry merge 后重建节点 polygon 与边几何
8. 最后回写有效节点-边关系，并 compact 成正式输出

这样对外正式对象始终只有 `node_info` 和 `edge_info`。

### 7.2 数据结构（概念）

- `DeadEndView`: `{point_rc, component_id}`
  - `point_rc` 断头路节点位置
  - `component_id` 断头路来源连通分量 id
- `InitialJunctionCandidateView`: `{candidate_id, member_points_rc, representative_point_rc}`
  - `candidate_id` 第 2 步内部交汇候选编号
  - `member_points_rc` 该候选对应的局部骨架像素集合
  - `representative_point_rc` 该候选压缩后的代表点
- `MergeGroupView`: `{survivor_node_id, member_node_ids, internal_edge_ids, external_edge_ids}`
  - `survivor_node_id` 合并后保留的节点 id
  - `member_node_ids` 被并入的节点 id 集合，包含 survivor 自身
  - `internal_edge_ids` 合并后变成组内边的 edge_id 集合
  - `external_edge_ids` 需要重定向的外连边 edge_id 集合
- `NodeZoneView`: `{node_id, point_rc, zone_mask}`
  - `node_id` 节点 id
  - `point_rc` 节点位置
  - `zone_mask` 节点占据区掩膜
- `ResidualEdgeComponentView`: `{component_id, pixels_rc, adjacent_node_ids}`
  - `component_id` 残余骨架连通分量 id
  - `pixels_rc` 连通分量像素集合
  - `adjacent_node_ids` 该分量接触到的节点集合
- `JunctionLocalView`: `{node_id, incident_edge_ids, local_mask, local_free_mask, old_centers_rc}`
  - `node_id` 当前交汇节点 id
  - `incident_edge_ids` 与该节点相连的边 id 集合
  - `local_mask` 节点局部骨架工作域
  - `local_free_mask` 节点局部自由空间工作域
  - `old_centers_rc` 当前节点对应的旧中心集合；合并节点与单节点在这里区分
- `PathSplitView`: `{edge_id, inner_path_rc, outer_path_rc, path_rc}`
  - `edge_id` 当前边 id
  - `inner_path_rc` 节点 polygon 内部路径
  - `outer_path_rc` 节点 polygon 外部路径
  - `path_rc` 最终完整路径
- `PostGeometryMergeCandidateView`: `{edge_id, src_node_id, dst_node_id, outer_path_length_m, full_path_length_m, outer_ratio, reason, action}`
  - `edge_id` 边切分后被判定为交汇内部短边的 edge id
  - `src_node_id / dst_node_id` 待合并的两端交汇节点
  - `outer_path_length_m` polygon 外部剩余边长，单位 m
  - `full_path_length_m` 完整边长，单位 m
  - `outer_ratio` 外部剩余边长占完整边长的比例
  - `reason` 命中二次合并或异常记录的原因，例如 `outer_path_too_short` 或 `long_edge_with_tiny_outer_path`
  - `action` 后续处理动作，`merge` 表示 A 类正常合并，`debug_anomaly` 表示 B 类异常记录
- `InactiveItemView`: `{item_type, item_id, inactive_reason, redirect_target_id}`
  - `item_type` `node | edge`
  - `item_id` 被置为无效的对象 id
  - `inactive_reason` 无效原因
  - `redirect_target_id` 若发生重定向，记录其目标对象 id

**与现有 contracts 的对应关系（逻辑视图映射）**
- `InitialJunctionCandidateView`
  - 对应第 2 步内部从骨架世界提炼出的初始交汇候选
  - 仅作第 2 步内部视图，不进入正式对象层
- `DeadEndView`
  - 对应当前研究代码中短枝修剪后压缩的断头路端点条目
  - 仅作第 2 步内部视图，不进入正式对象层
- `MergeGroupView`
  - 对应第 2 步内部的节点聚类/节点合并计划
  - 仅作第 2 步内部视图，不进入正式对象层
- `NodeZoneView`
  - 对应当前研究代码中 `node_zone_labels` 与节点占据区逻辑
  - 仅作建边阶段的内部视图
- `ResidualEdgeComponentView`
  - 对应当前研究代码中扣掉节点区后的残余骨架连通分量
  - 仅作建边阶段的内部视图
- `JunctionLocalView`
  - 对应当前 `07_5` 交汇重建时的节点局部工作域
  - 仅作交汇节点更新时的内部视图
- `PathSplitView`
  - 最终写回 `edge_info.inner_path_rc / outer_path_rc / path_rc`
- `PostGeometryMergeCandidateView`
  - 对应边切分后发现的交汇内部短边
  - 仅作二次节点合并的内部决策视图，不进入正式对象层
- `InactiveItemView`
  - 对应第 2 步内部的删除/失效记录
  - 正式输出前应被 compact，不进入正式对象层

### 7.3 输入输出说明

- 输入（对外）：`geometry_preparation_result`
  - 正式外部输入只有第 1 步输出的几何世界
- 输入（对内）：
  - `DeadEndView[]`
  - `InitialJunctionCandidateView[]`
  - `MergeGroupView[]`
  - `NodeZoneView[]`
  - `ResidualEdgeComponentView[]`
  - `JunctionLocalView[]`
  - `PostGeometryMergeCandidateView[]`
  - `InactiveItemView[]`
- 输出（对外）：`junction_rebuild_result`
  - `node_info_list`
  - `edge_info_list`
  - `debug_info`
- 输出（对内）：
  - 截断点
  - 扇区/支撑结构
  - polygon
  - 路径切分结果
  这些都只作为内部推理或 debug，不升级为正式对象

### 7.4 伪代码

#### 对外入口（接几何世界）

```text
function build_junction_rebuild(geometry_preparation_result, config):
    # 输入:
    #   geometry_preparation_result: 第 1 步输出的正式几何世界。
    # 输出:
    #   junction_rebuild_result。
    # 约束:
    #   本阶段只写 node_info / edge_info，不建立 graph_info。
    #   截断点、支撑结构、polygon 都只能作为更新 node/edge 的过程量。
    # 失败:
    #   只要 node/edge 闭环被破坏，必须立即报错。

    # 1) 先在基础骨架世界上建立初始 node_info / edge_info。
    # 这一步的目标不是得到最终正确几何，而是先把“哪些是节点、哪些是边”
    # 稳定落到正式对象层。后续所有交汇重建都只允许更新这些对象，不再新造
    # 一套并列对象来和 node_info / edge_info 竞争语义。
    node_info_list, edge_info_list = build_initial_node_edge(
        geometry_preparation_result,
        config,
    )

    # 2) 在初始节点上执行聚类与节点合并。
    # 这里解决的是“多个初始交汇节点其实属于同一个交汇结构”的问题。
    # 合并之后，后续几何求解只围绕保留节点进行；被吞并节点与无效边只保留
    # lineage/debug 语义，不再参与正式几何更新。
    merge_groups = derive_merge_groups(node_info_list, config)
    node_info_list, edge_info_list, inactive_items = apply_node_merges(
        node_info_list,
        edge_info_list,
        merge_groups,
        config,
    )

    # 3) 对保留交汇节点运行研究主线。
    # 是否为合并节点，只体现在 old_centers_rc 的数量和来源。
    # 研究算法原意里，这里直接进入：
    #   截断点 -> 扇区/支撑结构 -> polygon -> 必要时第二轮
    # 中间不额外插“局部骨架重建阶段”。
    node_info_list = rebuild_junction_nodes(
        node_info_list,
        edge_info_list,
        geometry_preparation_result,
        config,
    )

    # 4) 对 dead_end 节点建立最小节点占据区 polygon。
    # dead_end 不走交汇那套 support/polygon 主链，但仍需要一个最小节点边界，
    # 否则边在端点附近无法统一切出 inner / outer 路径。
    node_info_list = update_dead_end_polygon(
        node_info_list,
        config,
    )

    # 5) 用 hits / fallback bridge 主线重建有效边路径。
    # 这里必须显式保留研究算法里的两态逻辑：
    #   正常命中 -> 正常切分
    #   命中不全 -> fallback bridge
    # 不能再偷偷换成别的 heuristic 路线。
    edge_info_list = rebuild_edge_paths(
        edge_info_list,
        node_info_list,
        geometry_preparation_result,
        config,
    )

    # 6) 边切分后检查交汇内部短边。
    # 这一步专门处理“两个 junction polygon 几乎相接，导致 outer_path_rc
    # 极短或不存在”的情况。它不能下放到 coverage 阶段，否则会变成跳过
    # sweep 而不修复拓扑。
    post_geometry_merge_debug = []
    for iteration_index in range(config.max_post_geometry_merge_iterations):
        post_geometry_merge_candidates = derive_post_geometry_merge_candidates(
            node_info_list,
            edge_info_list,
            geometry_preparation_result.resolution_m_per_px,
            config,
        )
        post_split_geometry_anomalies = derive_post_split_geometry_anomalies(
            node_info_list,
            edge_info_list,
            geometry_preparation_result.resolution_m_per_px,
            config,
        )
        if post_split_geometry_anomalies and config.post_geometry_merge_strict_anomaly_fail:
            raise PostSplitGeometryAnomaly(post_split_geometry_anomalies)

        post_geometry_merge_debug.append(
            {
                "iteration_index": iteration_index,
                "merge_candidates": post_geometry_merge_candidates,
                "anomaly_findings": post_split_geometry_anomalies,
            }
        )
        post_merge_groups = derive_post_geometry_merge_groups(post_geometry_merge_candidates)
        if not post_merge_groups:
            # 没有 A 类 merge 候选时，本轮二次合并结束。
            # 若本轮存在 B 类 anomaly，它们已经进入 debug，不触发节点合并。
            break

        # 二次合并仍然复用节点合并的对象语义：
        #   两端 junction 合并
        #   命中短边变成内部 inactive edge
        #   外连边重定向到 survivor
        #   自环/重复边清理
        # 合并之后必须重新求 polygon 和边切分，因为 incident 关系和旧中心集合已改变。
        node_info_list, edge_info_list, post_inactive_items = apply_node_merges(
            node_info_list,
            edge_info_list,
            post_merge_groups,
            config,
        )
        inactive_items.extend(post_inactive_items)
        node_info_list, edge_info_list = refresh_node_edge_relations(node_info_list, edge_info_list)
        node_info_list = rebuild_junction_nodes(
            node_info_list,
            edge_info_list,
            geometry_preparation_result,
            config,
        )
        node_info_list = update_dead_end_polygon(
            node_info_list,
            config,
        )
        edge_info_list = rebuild_edge_paths(
            edge_info_list,
            node_info_list,
            geometry_preparation_result,
            config,
        )
        validate_post_geometry_merge_result(
            node_info_list,
            edge_info_list,
            post_geometry_merge_candidates,
        )
    else:
        # 达到迭代上限后必须先只读探测一次。
        # 只有仍存在 A 类 merge 候选时，才写 max-iteration warning。
        remaining_merge_candidates = derive_post_geometry_merge_candidates(
            node_info_list,
            edge_info_list,
            geometry_preparation_result.resolution_m_per_px,
            config,
        )
        if remaining_merge_candidates:
            post_geometry_merge_debug.append(
                {
                    "reason": "max_post_geometry_merge_iterations_reached_with_remaining_merge_candidates",
                    "max_iterations": config.max_post_geometry_merge_iterations,
                    "remaining_merge_candidates": remaining_merge_candidates,
                }
            )

    # 7) 回写有效节点和有效边的关系。
    # 第 2 步对外输出前，degree / incident_edge_ids 必须和边表完全一致，
    # 否则第 3 步建图阶段会直接建立在脏对象上。
    node_info_list, edge_info_list = refresh_node_edge_relations(node_info_list, edge_info_list)

    # 8) 正式输出前 compact，只保留有效对象。
    # compact 是第 2 步内部 active/inactive 语义和正式输出之间的边界。
    # 下游阶段只允许看到 compact 后的干净对象，不允许再消费 inactive 项。
    node_info_list, edge_info_list = compact_active_objects(node_info_list, edge_info_list)

    return {
        node_info_list,
        edge_info_list,
        debug_info = build_junction_debug_info(
            merge_groups,
            inactive_items,
            post_geometry_merge_debug,
            node_info_list,
            edge_info_list,
        ),
    }
```

```text
function derive_initial_nodes(geometry_preparation_result, config):
    # 1) 先从几何世界内部提炼初始交汇候选。
    # 这里的候选只是第 2 步内部视图，目的是从骨架世界中先抽出
    # “哪些像交汇”。它们还不是正式节点，只有压成 node_info 后才进入主线。
    initial_junction_candidates = derive_initial_junction_candidates(
        geometry_preparation_result,
        config,
    )

    # 2) 交汇节点来自初始交汇候选。
    # 每个候选压成一个正式 junction node，并在这里第一次分配 node_id。
    # 后续即使发生节点合并，也只是 inactive/redirect，不再重编号。
    node_info_list = []
    next_node_id = 1
    for junction in initial_junction_candidates:
        node_info_list.append(
            make_node_info(
                node_id = next_node_id,
                point_rc = junction.representative_point_rc,
                node_type = "junction",
                polygon_vertices_rc = None,
            )
        )
        next_node_id += 1

    # 3) 断头路节点来自短枝修剪后的端点压缩结果。
    # dead_end 在这里先进入正式对象层，原因是它们从一开始就参与建边，
    # 后续边切分也需要知道端点处是否有节点占据区。
    for endpoint in geometry_preparation_result.dead_end_items:
        node_info_list.append(
            make_node_info(
                node_id = next_node_id,
                point_rc = endpoint.point_rc,
                node_type = "dead_end",
                polygon_vertices_rc = None,
            )
        )
        next_node_id += 1

    return node_info_list
```

```text
function derive_initial_junction_candidates(geometry_preparation_result, config):

    return initial_junction_candidates
```

```text
function derive_initial_edges(
    rebuilt_skeleton_mask,
    node_info_list,
    resolution_m_per_px,
    config,
):
    # 1) 先在骨架上为每个节点建立占据区。
    # 这里的 zone 只用于把骨架切成“节点内部”和“节点外部主体链”。
    # 它不是最终 polygon，也不承担交汇几何求解职责。
    node_zone_views = build_node_zone_views(rebuilt_skeleton_mask, node_info_list, config)

    # 2) 从最终骨架中扣掉节点占据区。
    # 扣掉之后剩下的是节点之间的主体骨架分量，这正是初始 edge 的来源。
    residual_mask = subtract_node_zones_from_skeleton(rebuilt_skeleton_mask, node_zone_views)

    # 3) 提取残余骨架连通分量。
    # 每个连通分量都候选为一条初始边，但只有标准二端分量才能进入正式 edge。
    components = extract_connected_components(residual_mask)

    edge_info_list = []
    next_edge_id = 1
    for component in components:
        adjacent_node_ids = find_component_adjacent_nodes(component, node_zone_views)

        # 当前研究主线只接受标准二端边。
        # 若一个分量同时邻接 0/1/3+ 个节点，则它还不能被解释成正式边，
        # 应继续留在第 2 步内部调试视图，而不是强行写入 edge_info。
        if len(adjacent_node_ids) != 2:
            continue

        # 这里恢复的是残余骨架里的有序主体 path。
        # 后续 src/dst 节点区内部的 inner_path，不在这里求，而在 polygon 已知后再求。
        path_rc, src_contact_rc, dst_contact_rc = trace_component_path_rc(
            component,
            adjacent_node_ids,
            node_zone_views,
        )
        if len(path_rc) < 2:
            continue

        edge_info_list.append(
            make_edge_info(
                edge_id = next_edge_id,
                src_node_id = adjacent_node_ids[0],
                dst_node_id = adjacent_node_ids[1],
                inner_path_rc = [],
                outer_path_rc = path_rc,
                path_rc = path_rc,
                length_px = compute_path_length_px(path_rc),
                length_m = convert_px_to_m(compute_path_length_px(path_rc), resolution_m_per_px),
                src_contact_rc = src_contact_rc,
                dst_contact_rc = dst_contact_rc,
            )
        )
        next_edge_id += 1

    return edge_info_list
```

```text
function build_node_zone_views(rebuilt_skeleton_mask, node_info_list, config):
    # 1) junction 和 dead_end 的 zone 半径不同。
    # 这里的 zone 只是建边用的节点占据区，不是最终 polygon。
    # radius 的作用只是把节点中心从主骨架里“抠开”，因此可取小，
    # 不能把它误当成最终交汇几何尺度。
    views = []
    for node in node_info_list:
        zone_radius_px = (
            config.intersection_zone_radius_px
            if node.node_type == "junction"
            else config.endpoint_zone_radius_px
        )
        zone_mask = rasterize_disk_mask(
            rebuilt_skeleton_mask.shape,
            node.point_rc,
            zone_radius_px,
        )
        views.append(
            NodeZoneView(
                node_id = node.node_id,
                point_rc = node.point_rc,
                zone_mask = zone_mask,
            )
        )
    return views
```

```text
function find_component_adjacent_nodes(component_mask, node_zone_views):
    # 相邻关系只回答“这个残余分量接到了哪些节点区”，
    # 不回答路径方向，也不回答哪个是 src/dst。
    adjacent_node_ids = []
    for zone in node_zone_views:
        if masks_touch(component_mask, zone.zone_mask):
            adjacent_node_ids.append(zone.node_id)
    return adjacent_node_ids
```

```text
function trace_component_path_rc(component_mask, adjacent_node_ids, node_zone_views):
    # 先扣掉 src/dst 节点区内部像素，避免路径把节点内部小区也算进主体边链。
    src_zone = lookup_node_zone(adjacent_node_ids[0], node_zone_views)
    dst_zone = lookup_node_zone(adjacent_node_ids[1], node_zone_views)
    residual_mask = component_mask.copy()
    residual_mask[src_zone.zone_mask] = 0
    residual_mask[dst_zone.zone_mask] = 0

    # src/dst contact 是后续边重建的重要锚点：
    # 初始 edge 需要它来表达主体链是如何接到节点区边界上的。
    src_contacts = collect_contact_pixels(residual_mask, src_zone.zone_mask)
    dst_contacts = collect_contact_pixels(residual_mask, dst_zone.zone_mask)

    # 多源最短路恢复的 path 必须是有序的。
    # 如果这里拿不到有序 path，后续 edge.path_rc 就不具备可更新性。
    path_rc, src_contact_rc, dst_contact_rc = multi_source_shortest_path(
        residual_mask,
        src_contacts,
        dst_contacts,
    )
    return path_rc, src_contact_rc, dst_contact_rc
```

```text
function validate_initial_node_edge(node_info_list, edge_info_list):
    # 这里校验的是第 2 步最早的一层正式闭环。
    # 一旦这里不自洽，后面 merge、polygon、hits 全都会建立在错误对象之上。
    if len(node_info_list) == 0:
        error("initial nodes are empty")
    if len(edge_info_list) == 0:
        error("initial edges are empty")

    node_ids = set(node.node_id for node in node_info_list)
    for edge in edge_info_list:
        if edge.src_node_id not in node_ids:
            error("edge src node missing")
        if edge.dst_node_id not in node_ids:
            error("edge dst node missing")
        if len(edge.path_rc) < 2:
            error("edge path too short")
```

```text
function derive_merge_groups(node_info_list, config):
    # 这里只在 junction 节点之间做聚类。
    # dead_end 不参与聚类，因为它们不代表需要合并的交汇结构。
    junction_nodes = [node for node in node_info_list if node.node_type == "junction"]
    raw_groups = cluster_nearby_junction_nodes(junction_nodes, config)

    merge_groups = []
    for group in raw_groups:
        # survivor 暂时直接取 node_id 最小值，避免引入额外 heuristic。
        # 这里的关键目标是稳定和可复现，不是追求几何最优。
        survivor_node_id = min(node.node_id for node in group)
        merge_groups.append(
            MergeGroupView(
                survivor_node_id = survivor_node_id,
                member_node_ids = [node.node_id for node in group],
                internal_edge_ids = [],
                external_edge_ids = [],
            )
        )
    return merge_groups
```

```text
function apply_node_merges(node_info_list, edge_info_list, merge_groups, config):
    inactive_items = []

    for group in merge_groups:
        if len(group.member_node_ids) <= 1:
            continue

        survivor = lookup_node(group.survivor_node_id, node_info_list)
        member_ids = set(group.member_node_ids)

        # 1) 被合并节点先置为 inactive，而不是立刻物理删除。
        # 这样能保留 merge lineage，便于后续调试和错误追溯。
        for node_id in member_ids:
            if node_id == survivor.node_id:
                continue
            node = lookup_node(node_id, node_info_list)
            node.active = false
            node.merge_target_node_id = survivor.node_id
            inactive_items.append(
                InactiveItemView(
                    item_type = "node",
                    item_id = node.node_id,
                    inactive_reason = "merged_into_survivor",
                    redirect_target_id = survivor.node_id,
                )
            )

        # 2) 组内边直接失效；组外边重定向到 survivor。
        # 这一步解决的是“对象一致性”，不是最终几何。
        # 做完之后，保留节点和保留边在拓扑关系上必须已经统一。
        for edge in edge_info_list:
            if not is_edge_active(edge):
                continue
            src_in = edge.src_node_id in member_ids
            dst_in = edge.dst_node_id in member_ids

            if src_in and dst_in:
                edge.active = false
                edge.inactive_reason = "internal_after_node_merge"
                inactive_items.append(
                    InactiveItemView(
                        item_type = "edge",
                        item_id = edge.edge_id,
                        inactive_reason = "internal_after_node_merge",
                        redirect_target_id = None,
                    )
                )
                continue

            if src_in:
                edge.src_node_id = survivor.node_id
            if dst_in:
                edge.dst_node_id = survivor.node_id

        # 3) 清理重定向后产生的自环边和重复边。
        # 如果不清理，这些伪边会直接污染后面的交汇几何和边切分主链。
        edge_info_list, loop_items = deactivate_self_loops(edge_info_list)
        edge_info_list, dup_items = deactivate_duplicate_edges(edge_info_list)
        inactive_items.extend(loop_items)
        inactive_items.extend(dup_items)

    return node_info_list, edge_info_list, inactive_items
```

```text
function rebuild_junction_nodes(
    node_info_list,
    edge_info_list,
    geometry_preparation_result,
    config,
):
    for node in node_info_list:
        if node.node_type != "junction":
            continue
        if not is_node_active(node):
            continue

        local_view = build_junction_local_view(
            node,
            edge_info_list,
            geometry_preparation_result,
            config,
        )

        # 1) 是否为合并节点，只体现在 old_centers_rc 的数量和来源。
        #    单节点与合并节点走的是同一条主链，只是在“旧中心集合”这一输入上不同。
        # 2) 先找截断点，再找扇区/支撑结构，再解 polygon。
        # 3) 若 polygon centroid 与当前中心差异过大，再来一轮。
        #    这里的“再来一轮”是研究主线的一部分，不是额外 heuristic。
        node_geometry = solve_junction_geometry(local_view, config)

        node.point_rc = node_geometry.center_rc
        node.polygon_vertices_rc = node_geometry.polygon_vertices_rc
        node.debug_info["junction_geometry"] = node_geometry.debug_info

    return node_info_list
```

```text
function solve_junction_geometry(local_view, config):
    # 第 1 轮：用当前中心求截断点、支撑结构和 polygon。
    truncation_points = solve_truncation_points(local_view, config)
    support_elements = solve_support_elements(local_view, truncation_points, config)
    polygon_vertices_rc = solve_node_polygon(local_view, truncation_points, support_elements, config)
    center_rc = polygon_centroid(polygon_vertices_rc)

    # 若 polygon centroid 和当前中心偏差过大，说明支撑结构对中心有显著修正，
    # 这时需要用新中心再跑一轮，避免“中心仍停留在旧位置、polygon 已经变了”。
    if need_round2(center_rc, local_view.center_rc, config):
        local_view.center_rc = center_rc
        truncation_points = solve_truncation_points(local_view, config)
        support_elements = solve_support_elements(local_view, truncation_points, config)
        polygon_vertices_rc = solve_node_polygon(local_view, truncation_points, support_elements, config)
        center_rc = polygon_centroid(polygon_vertices_rc)

    # 这里不引入新的 truncation fallback。
    # 若 support/polygon 求解失败，应在 validate_polygon 或更早阶段显式报错。
    validate_polygon(polygon_vertices_rc)
    return {
        center_rc,
        polygon_vertices_rc,
        debug_info = {
            truncation_points,
            support_elements,
        },
    }
```

```text
function rebuild_edge_paths(edge_info_list, node_info_list, geometry_preparation_result, config):
    for edge in edge_info_list:
        if not is_edge_active(edge):
            continue
        src_node = lookup_node(edge.src_node_id, node_info_list)
        dst_node = lookup_node(edge.dst_node_id, node_info_list)

        # hits 只负责回答“path 与 src/dst polygon 的有效交点是什么”。
        # 真正的切分状态机必须集中在 split_path_by_hits(...) 里表达，
        # 不能在这里分散塞 heuristic。
        hits = intersect_path_with_node_polygons(edge.path_rc, src_node, dst_node)
        split = split_path_by_hits(edge.path_rc, hits, src_node, dst_node, config)

        edge.inner_path_rc = split.inner_path_rc
        edge.outer_path_rc = split.outer_path_rc
        edge.path_rc = split.path_rc
        edge.length_px = compute_path_length_px(edge.path_rc)
        edge.length_m = convert_px_to_m(edge.length_px, geometry_preparation_result.resolution_m_per_px)

    return edge_info_list
```

```text
function split_path_by_hits(path_rc, hits, src_node, dst_node, config):
    # 1) 若 src/dst 两侧都找到有效交点，则按交点正常切分。
    # 这时 inner / outer / full path 都是“正常几何语义”。
    if has_valid_hits_on_both_sides(hits):
        return build_split_from_hits(path_rc, hits)

    # 2) 若某一侧没有交点，则按当前主线保留 fallback bridge。
    # fallback bridge 本意不是更优几何，而是保证图结构不断。
    # 这里的 bridge 是研究主线明确定义的兜底，不应再被别的 polygon fallback 替代。
    return build_split_with_fallback_bridge(path_rc, hits, src_node, dst_node, config)
```

```text
function iter_post_geometry_edge_views(node_info_list, edge_info_list, resolution_m_per_px):
    views = []
    for edge in edge_info_list:
        if not is_edge_active(edge):
            continue

        src_node = lookup_node(edge.src_node_id, node_info_list)
        dst_node = lookup_node(edge.dst_node_id, node_info_list)
        if not is_node_active(src_node) or not is_node_active(dst_node):
            # 二次合并只能作用在当前 active 对象上。
            # 已被前序 merge 吞并的 inactive 节点不能再次参与重定向决策。
            continue

        if src_node.node_type != "junction" or dst_node.node_type != "junction":
            # dead_end 端不参与这类二次合并。
            # dead_end 短边可能代表真实末端，不应被误收成交汇内部结构。
            continue

        outer_path_length_m = compute_path_length_m(edge.outer_path_rc, resolution_m_per_px)
        full_path_length_m = compute_path_length_m(edge.path_rc, resolution_m_per_px)
        if full_path_length_m <= 0:
            continue

        outer_ratio = outer_path_length_m / full_path_length_m
        split_evidence = read_split_evidence(edge.debug_info)

        views.append(
            PostGeometryEdgeView(
                edge_id = edge.edge_id,
                src_node_id = src_node.node_id,
                dst_node_id = dst_node.node_id,
                outer_path_length_m = outer_path_length_m,
                full_path_length_m = full_path_length_m,
                outer_ratio = outer_ratio,
                split_evidence = split_evidence,
            )
        )

    return views
```

```text
function derive_post_geometry_merge_groups(post_geometry_merge_candidates):
    # 每个 merge candidate 都在两端 junction 之间加一条“应合并”关系。
    # post-split anomaly finding 单独走 debug/validation warning，不传入这个分组函数。
    # 若多条短边串成链，则统一按连通分量合成一个 merge group。
    adjacency = {}
    for candidate in post_geometry_merge_candidates:
        if candidate.action != "merge":
            continue
        adjacency[candidate.src_node_id].add(candidate.dst_node_id)
        adjacency[candidate.dst_node_id].add(candidate.src_node_id)

    groups = []
    for component in connected_components(adjacency):
        survivor_node_id = min(component)
        groups.append(
            MergeGroupView(
                survivor_node_id = survivor_node_id,
                member_node_ids = sorted(component),
                internal_edge_ids = [
                    candidate.edge_id
                    for candidate in post_geometry_merge_candidates
                    if candidate.action == "merge"
                    and candidate.src_node_id in component
                    and candidate.dst_node_id in component
                ],
                external_edge_ids = [],
            )
        )
    return groups
```

二次合并的判据应优先使用 `outer_path_rc` 的绝对长度和相对比例，而不是 `coverage_planning` 的 `effective_region_pixels`。
原因是：
- `junction_rebuild` 不应读取下游 coverage 结果
- `effective_region_pixels` 依赖 coverage 的自由区、端点 polygon block、territory 归属等额外规则
- `outer_path_rc` 是第 2 步自己刚刚产生的边切分真值，最能直接说明这条边是否仍是 polygon 外部通道
- 若需要调试，可在 `debug_info` 中同时记录候选边的 `outer_path_length_m / full_path_length_m / outer_ratio / reason / action`

```text
function refresh_node_edge_relations(node_info_list, edge_info_list):
    # 先清空再回填，避免保留 merge 前或切分前的脏 incident 信息。
    for node in node_info_list:
        if not is_node_active(node):
            continue
        node.incident_edge_ids = []

    # edge 表才是节点 incident 关系的单一真值来源。
    for edge in edge_info_list:
        if not is_edge_active(edge):
            continue
        lookup_node(edge.src_node_id, node_info_list).incident_edge_ids.append(edge.edge_id)
        lookup_node(edge.dst_node_id, node_info_list).incident_edge_ids.append(edge.edge_id)

    # degree 是 incident_edge_ids 的派生字段，必须在最终阶段统一刷新。
    for node in node_info_list:
        node.incident_edge_ids = deduplicate_preserve_order(node.incident_edge_ids)
        node.degree = len(node.incident_edge_ids)

    return node_info_list, edge_info_list
```

```text
function compact_active_objects(node_info_list, edge_info_list):
    # compact 只做“对象层收口”，不再做新的几何推理。
    active_nodes = [node for node in node_info_list if is_node_active(node)]
    active_edges = [edge for edge in edge_info_list if is_edge_active(edge)]

    active_node_ids = set(node.node_id for node in active_nodes)
    # 悬空边不能进入正式输出；否则第 3 步 graph_info 会天然带坏边。
    active_edges = [
        edge
        for edge in active_edges
        if edge.src_node_id in active_node_ids and edge.dst_node_id in active_node_ids
    ]

    return active_nodes, active_edges
```
