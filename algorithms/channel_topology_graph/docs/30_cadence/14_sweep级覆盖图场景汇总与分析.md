# sweep级覆盖图场景汇总与分析

目标：在正式设计 `sweep` 级覆盖图与顺序规划之前，先把会遇到的场景梳理清楚，形成稳定的问题空间。  
本文档重点回答：

- 会遇到哪些类型的路口
- 会遇到哪些类型的通道
- 哪些执行风险会直接影响 `sweep -> sweep` 的连接
- 后续测试用例应该围绕哪些典型场景构造

本文档不直接给最终算法，只负责：

- 场景汇总
- 场景特征分析
- 难点归类

细化规则与正式方案请继续参考：

- [15_sweep级覆盖图特征提取与映射思路.md](./15_sweep级覆盖图特征提取与映射思路.md)
- [16_sweep级覆盖图与顺序规划方案.md](./16_sweep级覆盖图与顺序规划方案.md)

## 1. 问题背景

当前第 3 步已经给出了：

- edge 级拓扑许可
- 即 accepted `in_edge -> via_node -> out_edge`

当前第 4 步前半段已经给出了：

- `coverage_lane`
- `sweeps`
- 单条 `sweep` 上的 `SweepNodeChain`

接下来真正要解决的问题是：

- 如何在节点处建立 `sweep -> sweep` 的连接关系
- 如何在 `sweep` 级别形成稳定的覆盖图
- 如何在该图上得到稳定的覆盖顺序

这里的核心难点不是“有没有通路”，而是：

- 同一节点内会存在多条 edge
- 同一 edge 内又会存在多条 sweep
- 若处理不当，组合规模会爆炸
- 若映射策略不稳定，route 会在局部抖动、碰撞风险上升、路口区覆盖变差

## 2. 场景分类原则

场景不能只按处理流程分类，而应按结构与风险分类。

本文档按三条主线分类：

1. 路口类型
2. 通道类型
3. 运动风险类型

这样后续：

- 测试用例能按结构覆盖
- 映射策略能按特征选模板
- 方案文档不会退化成“见招拆招”

## 3. 路口类型场景

### 3.1 I 型连续通道

特征：

- 实际上是同一主通道连续通过
- 节点只是几何上被识别出来，但没有真正分支选择

意义：

- 这是最简单场景
- 更接近“通道连续延伸”
- 可以作为 `straight` 映射基线

主要风险：

- 不应把它过度复杂化成真正多分支路口

### 3.2 L 型拐角路口

特征：

- 两条 edge 成拐角连接
- 没有直行 through
- 只有转弯

意义：

- 是最基础的转向场景
- 用来分析左/右转的最小映射问题

主要风险：

- 外侧 sweep 更容易靠近障碍
- 小转弯半径风险更高

### 3.3 T 型三岔路口

特征：

- 一个主干，一个支路
- 同时存在：
  - 主干直行
  - 主干转入支路
  - 支路并入主干

意义：

- 是最常见的多选择节点
- 可用来分析：
  - 直行优先
  - 支路接入
  - 左右转差异

主要风险：

- 不同 `edge-pair` 的 sweep 映射规则可能不同
- 支路通常 sweep 数更少、更窄

### 3.4 X 型四向交汇

特征：

- 四向 edge 汇合
- 同时存在：
  - 直行
  - 左转
  - 右转

意义：

- 是最核心、最复杂的标准路口
- 适合验证 sweep 级连接策略是否稳定

主要风险：

- 映射模板多
- 候选多
- 路口区覆盖收益的影响也更明显

### 3.5 Y 型 / 不对称三叉

特征：

- 三个分支存在明显不等夹角
- 节点口不是正交关系

意义：

- 这是索引映射最容易失真的场景
- 用来验证“不能只看索引”的必要性

主要风险：

- 同序/反序不再天然成立
- 必须更多依赖节点口空间排序和转向角

### 3.6 环岛 / 开阔回环 / 特殊开放节点

特征：

- 节点内部不是普通局部连接区
- 可能存在局部环形自由区或大开阔连接区

意义：

- 不适合直接套普通 `L/T/X/Y` 模板
- 应作为特殊节点类型保留

主要风险：

- 若强行套普通路口策略，组合会失真
- 路口区覆盖收益与普通节点不同

## 4. 通道类型场景

### 4.1 单 sweep 通道

特征：

- 通道宽度只容纳 1 条 sweep

意义：

- 连接映射最简单
- 可作为 sweep 级连接的下界场景

主要风险：

- 通道端部容易受节点口约束
- 一旦连接不通就没有冗余 sweep 可替代

### 4.2 多 sweep 通道

特征：

- 通道可容纳 2 条及以上 sweep

意义：

- 这是 sweep 级覆盖图的主战场
- 需要定义组内顺序、中心/边缘语义、连接策略

主要风险：

- 不能做全排列
- 必须建立稳定的组内结构与组间映射

### 4.3 宽度变化通道

特征：

- 同一通道内部可用横向区间变化明显
- 不同锚点的局部可铺 sweep 数不同

意义：

- 对应当前横向铺设方案的核心场景
- 可用来验证稳健分位数选条数是否合理

主要风险：

- 通道整体 `sweep_count` 和局部可行性可能冲突
- 某些 sweep 在局部可能更靠边、更危险

### 4.4 短通道 / stub / 袋口通道

特征：

- 通道长度短
- 常常连接到大节点或边界口

意义：

- 可能需要保底策略
- 不宜强行套完整 sweep 族逻辑

主要风险：

- sweep 点链太短
- 连接选择非常敏感

## 5. 运动风险类型

### 5.1 小转弯半径风险

特征：

- 某条 `in_sweep -> out_sweep` 从几何上可通
- 但转弯过急

影响：

- 执行器风险高
- 连接段可能不稳定

### 5.2 边缘 sweep 风险

特征：

- 位于通道边缘的 sweep 更靠障碍

影响：

- 在节点口附近更容易碰撞
- 即使拓扑可连，也未必应优先选

### 5.3 通道内切换风险

特征：

- 在同一通道内直接从一条 sweep 切到相邻 sweep

影响：

- 并不一定优于借路口换向
- 有时中间跨一个、或借路口掉头反而更稳

### 5.4 路口区覆盖收益

特征：

- 某种映射虽然可通，但会让路口区留下较大未覆盖残区

影响：

- 说明连接策略不能只看可通行性
- 还要把“路口区附带覆盖收益”纳入评分

## 6. 典型组合场景

### 6.1 单 sweep 对单 sweep through

作用：

- 验证最小 sweep 图是否成立

### 6.2 多 sweep 对多 sweep straight

作用：

- 验证组内顺序是否稳定
- 验证同序/反序哪种更合理

### 6.3 多 sweep 对少 sweep merge / split

作用：

- 验证数量不等场景
- 验证中心对齐与端部补偿

### 6.4 多 sweep 转弯进窄支路

作用：

- 验证边缘 sweep 风险
- 验证转向类型对映射的影响

### 6.5 Y 型路口非正交转向

作用：

- 验证“不能只看索引映射”

### 6.6 路口掉头替代通道内切换

作用：

- 验证“路口掉头”是否可作为更优原语

## 7. 对测试用例设计的启发

后续测试用例建议围绕以下维度组合：

1. 节点类型
- `I/L/T/X/Y/special`

2. 通道 sweep 数
- `1/2/3/4+`

3. sweep 数是否对称
- 对称
- 不对称

4. 转向类型
- `straight`
- `left_turn`
- `right_turn`
- `u_turn`

5. 风险类型
- 近障碍
- 小转弯半径
- 边缘 sweep

## 8. 本文结论

当前问题不能再简单理解成：

- “给一个节点内所有 sweeps 做连接”

而应理解成：

- 不同结构路口
- 不同 sweep 数通道
- 不同风险模式

共同决定的 sweep 级连接与顺序问题。

因此下一步必须先做：

- 场景特征提取
- sweep 组结构定义
- 节点口空间排序
- 映射策略模板

而不是直接进入几何连接求解。
