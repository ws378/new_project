# 像素语义收口改造清单

## 1. 目标

本轮只收口一种问题：**像素单位已经离开几何内部层，进入正式对象、排序、打分、候选选择、节拍选择之后，语义会随着分辨率变化而漂移。**

这里不追求把所有 `px` 都删掉，而是做三件事：

1. 几何内部计算允许继续使用 `px`
2. 正式对象里凡是会被主线决策直接消费的尺度语义，统一改成 `m`
3. 主消费链禁止再直接读取这批 `px`

## 2. 收口原则

### 2.1 允许保留 `*_px` 的位置

只允许留在下面两类地方：

1. **几何内部工作量纲**
   - 例如采样步长、法向搜索半径、净空距离、局部偏移序列
   - 这些量本来就是在像素网格上求解，保留 `px` 合理
2. **纯渲染 / 纯调试 / 纯局部工作缓存**
   - 例如线宽、圆半径、裁剪留白、局部距离变换图
   - 这些不进入主线排序和候选裁决

### 2.2 必须改成 `*_m` 的位置

只要一个尺度量进入以下任一语义，就必须改成 `m`：

1. `sweep` 正式对象中的“相对中心偏移”
2. `sweep_transition_candidate` 正式对象中的“端点距离”
3. cadence / route repair / merge / reachability / greedy 的排序、评分、阈值、代理成本
4. 任何会跨 case、跨分辨率比较的正式 summary / candidate / route 字段

### 2.3 必须禁止直接读 `px` 的消费点

下面这些消费面必须只读 `m`：

1. `sweep_graph` 里按横向顺序对 `sweep` 排序
2. `sweep_transition_candidate` 的 `total_score`
3. cadence 起点选择
4. cadence greedy 动作排序
5. cadence 可达性代理成本
6. route merge / absorb / repair 的 transition 优先级比较

## 4. 公共换算口径

新增统一 helper，禁止在消费层手写：

- `px_to_m(...)`
- `m_to_px(...)`
- `sequence_px_to_m(...)`

要求：

1. 正式对象写盘前统一换算
2. cadence / candidate / ranking 只能消费 `m`
3. 若几何内部仍需 `px`，必须在局部算完后立即在边界处转成 `m`

## 5. 本轮不动的 `px`

下面这些 `px` 目前仍保留，原因是它们还属于几何 / 渲染 /局部工作量纲，而不是主线决策语义：

1. `geometry_preparation/*` 内部骨架、pruning 距离
2. `junction_rebuild/*` 内部局部几何半径、cut/probe/verify 距离
3. `final_coverage_path/final_path_connectors.py` 内部局部距离变换、采样步长、support-band 光栅宽度
4. `renderers/*` 里的可视化线宽、留白、圆半径
5. `path_length_px` 这类纯统计/报告字段，只要没有被 cadence / candidate / topology 主线直接消费，就暂时允许保留

## 6. 实施顺序

1. 先补公共换算 helper
2. 先改 `SweepInfo` 正式对象
3. 再改 `SweepTransitionCandidateItem` 正式对象
4. 再改 `sweep_graph` / `cadence` 全部主消费点
5. 最后改测试与真实 case 验证
