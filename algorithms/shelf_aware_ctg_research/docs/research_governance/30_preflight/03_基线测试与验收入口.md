# 基线测试与验收入口

## 1. 通用检查

每批开始和结束：

```bash
git status --short
git diff --stat
```

术语残留检查：

- 对研究目录搜索独立 axis path 旧关键词，`output/` 除外。
- 命中必须为 0。
- 允许保留 edge axis / axis grid / 无向 axis 这类辅助信息表述。

## 2. 文档批次验收

适用：新增或修改 governance / docs。

检查：

```bash
git diff -- algorithms/shelf_aware_ctg_research/docs
find algorithms/shelf_aware_ctg_research/docs/research_governance -maxdepth 3 -type f | sort
```

通过标准：

- README 已挂接新增文档。
- 文档没有把待验证判断写成事实。
- 没有恢复独立 axis path 口径。

## 3. baseline run 验收

命令：

```bash
env PYTHONDONTWRITEBYTECODE=1 python3 -m algorithms.shelf_aware_ctg_research.scripts.run_project_study \
  --project-dir examples/maptools_projects/fourfloor \
  --project-dir examples/maptools_projects/beiguo_lanshan_0407
```

检查：

```bash
RUN=algorithms/shelf_aware_ctg_research/output/run_<timestamp>
find "$RUN" -maxdepth 1 -mindepth 1 -type d | wc -l
find "$RUN" -maxdepth 2 -name "09_shelf_aware_baseline_path_overlay.png" | wc -l
python3 - <<'CHECK_LEGACY_TERMS'
from pathlib import Path
run = Path("$RUN")
legacy = ["shelf_aware_" + "axis", "axis_" + "prior", "path_point_" + "delta", "path_length_" + "delta"]
for path in run.rglob("*"):
    if path.is_file():
        text = path.read_text(errors="ignore")
        if any(token in text for token in legacy):
            print(path)
CHECK_LEGACY_TERMS
```

通过标准：

- area 数为 12。
- baseline overlay 数为 12。
- run 内没有独立 axis path 字段或目录。

## 4. B/C/D 诊断验收

命令：

```bash
env PYTHONDONTWRITEBYTECODE=1 python3 -m algorithms.shelf_aware_ctg_research.scripts.diagnose_path_grid_jumps \
  --run-dir algorithms/shelf_aware_ctg_research/output/run_<timestamp>
```

通过标准：

- 每个 area 有 diagnostics 目录。
- 每个 area 至少有 path-by-territory、grid-node-quality、jump-overlay 图。
- summary 只包含 `shelf_aware_baseline`。
- 重点区域 `beiguo_lanshan_0407_area_6`、`fourfloor_area_6`、`fourfloor_area_7` 有可解释输出。

## 5. 新诊断脚本验收

新增 B/C/D 加强诊断时，最低检查：

```bash
python3 -m compileall algorithms/shelf_aware_ctg_research
env PYTHONDONTWRITEBYTECODE=1 python3 -m algorithms.shelf_aware_ctg_research.scripts.<new_script> \
  --run-dir algorithms/shelf_aware_ctg_research/output/run_<timestamp>
```

通过标准：

- 输出图和 JSON 数量与 area 数一致，或明确只针对重点 area。
- JSON 能追溯到 path point、grid node、jump segment。
- 图例能区分 free、obstacle、territory、junction、jump、candidate nodes。
- 不修改正式 planner 输出。

## 6. 研究版优化验收

进入 F 阶段后，必须至少比较：

- baseline path length。
- research path length 或 simulated path length。
- jump count。
- long jump count。
- optional / pruned node count。
- estimated coverage loss。

通过标准不能只看路径变短，还必须看覆盖损失和局部图。
