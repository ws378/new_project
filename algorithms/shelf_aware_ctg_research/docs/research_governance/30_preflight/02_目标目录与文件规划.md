## 2. 后续允许新增

### 2.1 诊断脚本

```text
scripts/diagnose_residual_grid_nodes.py
scripts/diagnose_jump_target_context.py
scripts/diagnose_grid_node_metrics.py
```

可以合并成一个脚本，但输出必须清楚区分 B/C/D。

### 2.2 诊断源码模块

```text
src/path_diagnostics.py
src/grid_node_diagnostics.py
src/jump_diagnostics.py
src/diagnostic_visualization.py
```

只有当脚本复杂度明显上升时才拆模块。

### 2.3 研究版优化模块

```text
src/residual_node_classification.py
src/research_pruning.py
scripts/run_residual_pruning_experiment.py
```

这些模块只能影响研究输出，不改正式 planner。

### 2.4 输出目录

每个 run 下按 area 输出：

```text
<project>_area_<id>/
  diagnostics/
    path_territory_jumps/
    grid_node_quality/
    residual_jump_targets/
    pruning_experiment/
```

## 3. 禁止落位

禁止新增或修改长期能力到：

- `algorithms/coverage_planning/planners/shelf_aware_guarded/`，除非用户明确进入正式 planner 优化。
- `algorithms/channel_topology_graph/`，除非当前批次是 CTG 本体修复。
- `coverage_dataset/`，当前研究默认不使用 business crop。
- `/tmp`，默认输出必须在 `algorithms/shelf_aware_ctg_research/output/`。

## 4. 输出提交规则

默认不提交：

- run 目录。
- PNG 大图。
- planner artifacts。

允许提交：

- 源码。
- 脚本。
- 文档。
- 小型固定 JSON summary，前提是它被明确作为文档证据，而不是临时 run 输出。
