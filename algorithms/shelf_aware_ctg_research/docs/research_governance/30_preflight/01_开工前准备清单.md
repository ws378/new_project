# 开工前准备清单

## 1. 目标

保证后续 B/C/D/E/F/G 研究可以持续推进，不需要每一步都重新讨论目录、输入、验收和提交边界。

## 2. 每批开工前必须确认

### 2.1 阶段

必须明确当前批次属于：

```text
A baseline 固定
B path + territory + jump 全局诊断
C grid node 质量诊断
D 残留点与远跳原因分析
E optional / snap / insert 判据
F 研究版 pruning / residual handling
G territory / axis 接入评估
```

### 2.2 范围

必须写清：

- 本批改哪些脚本。
- 本批新增哪些图或 JSON。
- 本批是否改 docs。
- 本批是否允许改 `src/`。
- 本批是否允许改正式 planner。

默认：不允许改正式 planner。

### 2.3 输入

默认输入：

```text
examples/maptools_projects/fourfloor
examples/maptools_projects/beiguo_lanshan_0407
```

默认 baseline run：

```text
algorithms/shelf_aware_ctg_research/output/run_20260505_185541_861008
```

如果重新生成 run，必须记录新 run path 和 area 数。

### 2.4 非范围

每批默认非范围：

- 不恢复独立 axis path。
- 不做 baseline vs 独立 axis path 对比。
- 不修改正式 `shelf_aware_guarded`。
- 不提交 output 大图。
- 不碰与本研究无关的 maptools project 数据改动。

### 2.5 工作区

开工前必须执行：

```bash
git status --short
```

处理原则：

- 用户已有改动不回退。
- 无关改动不提交。
- 同文件冲突先读 diff。

### 2.6 术语检查

提交前必须确认研究目录不出现独立 axis path 旧口径。检查方式：对旧关键词做全目录搜索，`output/` 除外，命中必须为 0。

旧关键词包括：独立 axis 路径目录名、旧路径差值字段、旧路径长度差值字段、以及英文旧称。

允许存在的 axis 表述只能是辅助信息、edge axis grid 或无向 axis 语义。
