# 资产与输入清单

## 1. 研究目录

```text
algorithms/shelf_aware_ctg_research/
  README.md
  docs/
  scripts/
  src/
  output/
```

## 2. 当前源码资产

```text
src/project_inputs.py
src/ctg_territory_extractor.py
src/boundary_smoothing.py
src/territory_expansion.py
src/direction_grid.py
src/visualization.py
src/study_runner.py
```

职责：

- `project_inputs.py`：读取 maptools project 与 area。
- `ctg_territory_extractor.py`：复用 CTG 前几步输出 geometry / junction / graph / lane info。
- `territory_expansion.py`：由 territory seed 生成 expanded territory label。
- `direction_grid.py`：由 edge outer path 生成 1m 无向 axis 样本。
- `visualization.py`：输出 CTG / territory / axis 辅助图。
- `study_runner.py`：生成 baseline path 与 summary。

## 3. 当前脚本资产

```text
scripts/run_project_study.py
scripts/diagnose_path_grid_jumps.py
```

当前状态：

- `run_project_study.py` 只生成 `shelf_aware_guarded` baseline path。
- `diagnose_path_grid_jumps.py` 只分析 baseline。
- 任何新增诊断脚本不得恢复独立 axis path。

## 4. 固定输入项目

当前研究默认输入：

```text
examples/maptools_projects/fourfloor

```

```text
coverage_dataset/cases/channel_topology_graph/business_maps/.../crop_S_xx
```

## 5. CTG 读取字段

当前辅助信息读取：

```text
coverage_lane_info[*].source_edge_id
coverage_lane_info[*].territory_pixels
graph_info.edges[*].outer_path_rc
graph_info.nodes[*].polygon_vertices_rc
geometry_result.free_mask
geometry_result.region_mask
```

当前不作为主输入：

```text
coverage_lane_info[*].sweep_ids
```

## 6. 输出资产口径

默认输出目录：

```text
algorithms/shelf_aware_ctg_research/output/run_<timestamp>/
```

输出默认不提交 Git。需要长期保留的结论写入 docs；需要长期保留的小型 summary 可另行固化为文档或 fixture。
