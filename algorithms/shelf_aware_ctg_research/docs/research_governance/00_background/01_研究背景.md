# 研究背景

## 1. 问题来源

当前 `shelf_aware_guarded` 在货架、墙边和局部边界细节上表现较好，但它缺少全局拓扑语义。典型问题包括：

- 回字形走廊、十字形道路、L 形区域中，单个全局主方向不能代表全区域。
- CTG edge / territory 能表达区域归属，但原始 edge 可能过碎，不能直接作为强约束。
- 当前覆盖网格节点一次性生成，后续不能删除、合并、吸附或插入。
- planner 可能为了一个或少量低价值残留节点产生远距离 fallback jump。
- 实际清扫宽度通常大于 nominal coverage width，部分残留点可能已经被路径重叠覆盖。

## 2. 已确认方向

当前认可的研究方向是：

```text
保留 shelf_aware_guarded 作为主体 planner
CTG / expanded territory / edge axis 只作为辅助图层、诊断证据和后续模块输入
先诊断 grid node 与 jump，再做模块优化
```

## 3. 不再继续的方向

以下内容不作为后续主线：

- 不让 axis 独立形成全局路径。
- 不再用独立 axis 路径的路径长度、jump 数量评价研究方向。
- 不直接把 axis 替代 baseline local direction。
- 不直接做 CTG-guided 组合算法。
- 不直接把原始 edge label 放进 energy 做强约束。
