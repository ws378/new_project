# 研究架构与边界

## 1. 总体架构

```text
maptools project area
  -> CTG auxiliary extraction
  -> expanded territory label map
  -> edge-axis direction grid
  -> shelf_aware_guarded baseline path
  -> diagnostics
  -> optional / snap / insert 判据研究
  -> 研究版 grid node pruning 或 residual handling
```

## 2. 主体与辅助关系

主体：

```text
shelf_aware_guarded baseline
```

辅助：

```text
expanded territory
edge axis grid
CTG graph/node/edge metadata
path jump diagnostics
grid node diagnostics
```

辅助信息只回答：

- 当前点属于哪个 territory。
- 当前点是否在 junction / unknown / connector。
- jump 是否跨 territory。
- node 是否靠边、低 degree、小面积、可能被 overlap 覆盖。
- local direction 后续是否需要 axis 解释或补充。

辅助信息当前不做：

- 不接管规划。
- 不替代 local direction。
- 不给出独立全局 path。
- 不把原始 edge label 强行变成覆盖顺序。

## 3. 研究模块边界

### A. baseline 固定模块

职责：固定输入、baseline 输出和基本 summary。

禁止：改 `shelf_aware_guarded` 正式算法。

### B. path + territory + jump 全局诊断

职责：把 path、territory、junction、jump 放到同一语义图中。

禁止：根据诊断结果直接删 path 或改 planner。

### C. grid node 质量诊断

职责：分析 grid node degree、cell free area、贴边、center adjustment、孤立/半孤立状态。

禁止：在未形成规则前删除节点。

### D. 残留点与远跳原因分析

职责：解释 jump target 为什么被访问，是否低价值，是否可由 overlap 覆盖。

禁止：只凭单张图判断所有区域都可删点。

### E. 判据形成

职责：形成 `required / optional_by_overlap / optional_by_small_area / snap_candidate / insert_candidate` 分类。

禁止：把判据写成已验证优化效果。

### F. 研究版 pruning / residual handling

职责：只在本研究目录实现实验版，不进正式 planner。

禁止：影响 `algorithms/coverage_planning/planners/shelf_aware_guarded` 的正式行为。

### G. territory / axis 接入评估

职责：在诊断证据充分后决定是否进入 energy、fallback 或 node 分类。

禁止：未经诊断直接恢复 axis 融合。

## 4. 硬边界

- 不修改正式 planner，除非用户明确进入正式优化阶段。
- 不提交 output 大图和批量 run 目录。
- 不碰用户已有 maptools project 数据改动。
- 不使用 `/tmp` 作为默认输出。
- 不把 `coverage_dataset` business crop 纳入当前默认输入。
