# 主题说明

`01_主题` 保存长期主题真值。这里的文档回答“系统应该是什么”，而不是“某次推进做了什么”。

## 职责

- 定义算法入口与能力边界；
- 定义 `shelf_aware_turn_cost` 的算法设计、数据结构、流程、诊断和禁止事项；
- 定义机器人几何、清扫 footprint、转角窗口和只读几何诊断契约。

## 文件组织

| 文件或目录 | 职责 |
| --- | --- |
| `01_算法入口与能力边界.md` | 说明 UI / formal planner 中每个算法 mode 的身份、能力边界和禁止误用。 |
| `02_shelfAwareTurnCost算法设计/` | `shelf_aware_turn_cost` 长期算法设计文档集。 |
| `03_机器人几何与覆盖定义/` | 后轴参考点、车体 footprint、清扫 footprint 和几何诊断定义。 |

## 禁止内容

- 不写执行流水；
- 不写临时计划；
- 不保存旧实验原文；
- 不把单个 case 的图像结论当作长期算法原则。
