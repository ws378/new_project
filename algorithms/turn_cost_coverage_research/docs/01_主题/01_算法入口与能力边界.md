# 算法入口与能力边界

本文是 `turn_cost_coverage_research` 当前算法入口的长期主题真值。它只说明“哪些算法入口可以被 UI 或 formal planner 使用，以及每个入口的边界是什么”。

本文不记录执行流水，不保存旧实验过程，不替代 `02_shelfAwareTurnCost算法设计/` 的详细算法设计。

## 1. 当前正式入口

| 展示名称 | mode | 类型 | 当前职责 |
| --- | --- | --- | --- |
| `基础算法` | `basic` | formal | 基础覆盖算法入口。 |
| `shelfAware` | `shelf_aware` | baseline | 当前保守正式基线。 |
| `ShelfAware+TurnCost` | `shelf_aware_turn_cost` | candidate | UI 可显式选择的候选增强入口。 |
| `通道拓扑图` | `channel_topology_graph` | adapter | 通道拓扑 adapter 入口，不属于 `planner_factory` formal mode。 |
| `自动选择` | `auto` | router | 当前不自动选择 `shelf_aware_turn_cost`。 |

## 2. `shelf_aware` 边界

`shelf_aware` 是当前保守正式基线。它的含义是：

- 由 `ShelfAwareCoveragePlanner` 生成路径；
- 使用 UI 当前保存的通用参数和 shelfAware 参数；
- CTG auxiliary 只作为已有 shelfAware 运行链路中的辅助能力；
- 质量守卫只做诊断或保护，不把 turn-cost 研究后处理接入路径；
- 不再保留独立的 “ShelfAware 原版” 或 “ShelfAware UI 稳定版” 对外选项。

`shelf_aware` 不等于：

- `shelf_aware_turn_cost`；
- official turn-cost planner；
- 研究脚本中的 lane shift、batch reconnect、shortcut 或节点吸附输出。

## 3. `shelf_aware_turn_cost` 边界

`shelf_aware_turn_cost` 是当前候选增强入口。它的当前工程定义是：

```text
ShelfAwareCoveragePlanner 主路径组织
+ turn_cost_repaired_grid 节点生成
+ repaired_grid_max_offset_factor=0.28
+ CTG auxiliary on
+ isolated jump cleanup off
```

它迁入的 turn-cost 研究价值只有当前已经进入主流程的部分：

- 规则覆盖点生成节拍；
- 受控补点修复；
- 候选评分中的转角代价；
- 局部方向和 CTG 辅助评分；
- 路径来源可追溯；
- 轻量质量摘要。

线间距稳定性、几何只读风险和效果守卫是当前验收或诊断关注项；它们不等同于已经完整进入主流程的 turn-cost 算法模型。lane family、topology source 和 coverage contribution 仍属于后续缺口。

它没有迁入的内容：

- 论文 official 完整流程；
- square8 / axis-guided official flow；
- HiGHS/Gurobi official 复现结果；
- batch reconnect 后处理；
- lane family 简单平移或重建；
- 全路径节点吸附；
- 删点、shortcut、局部重连研究输出；
- 几何只读诊断硬约束。

## 4. 正式能力边界

当前正式能力边界是：

```text
shelf_aware           = baseline
shelf_aware_turn_cost = UI-ready candidate mode
```

`shelf_aware_turn_cost` 可以进入 UI 显式选择和灰度使用，但不应直接替代 `shelf_aware` 成为无风险默认主线。

判断依据见：

- `docs/05_验证/当前基线/20260523_shelfAware基线.md`
- `docs/05_验证/当前基线/20260617_shelfAwareTurnCost候选基线.md`
- `docs/05_验证/01_shelfAwareTurnCost最终收敛验收清单.md`
- `docs/03_缺口/01_当前缺口总表.md`

## 5. UI 和自动选择规则

UI 可以展示：

- `shelfAware`
- `ShelfAware+TurnCost`

UI 不应展示：

- `ShelfAware 原版`
- `ShelfAware UI 稳定版`
- `official turn-cost`
- `square8 official flow`
- `batch reconnect`
- `lane family`
- `node snap`

`auto` 当前不自动路由到 `shelf_aware_turn_cost`。如果后续要允许自动选择，必须先补齐：

- 更大范围多项目/多区域门禁；
- GUI 交互级自动化或等价人工验收；
- 运行时预算和失败降级策略；
- 几何风险只读诊断的可读性和证据链；
- 明确的回滚条件。

## 6. 研究资产与正式入口的关系

研究脚本和 output 可以继续存在，但它们不是 formal planner 能力。

| 类别 | 当前作用 | 是否可直接进入 UI |
| --- | --- | --- |
| `scripts/baseline/*` | 基线和门禁支持。 | 仅通过 formal mode 间接体现。 |
| `scripts/diagnostics/*` | 只读诊断。 | 不直接改路径。 |
| `scripts/experiments/*` | 研究证据。 | 否。 |
| `scripts/archived/*` | 归档或拒绝路线复现。 | 否。 |
| `src/diagnostics/*` | 只读诊断实现。 | 只能输出诊断。 |
| `src/experiments/*` | 研究实现。 | 否。 |
| `src/guidance/*` | 方向引导研究参考。 | 否。 |
| `src/official_replacements/*` | official 复现替代参考。 | 否。 |

## 7. 禁止事项

- 不把 research experiments import 到 formal planner。
- 不把 batch reconnect / shortcut / lane shift / node snap 接成默认行为。
- 不把 raw `path_generation_provenance.json` 当成 final path provenance。
- 不把 `metadata.json` 升级成 formal 事实中心。
- 不把几何只读诊断直接作为 planner 硬约束。
- 不新增 turn-cost 私有预处理语义。
- 不为单个 area 写专用补丁。
- 不用 output 中某个单 run 反向定义算法入口。
