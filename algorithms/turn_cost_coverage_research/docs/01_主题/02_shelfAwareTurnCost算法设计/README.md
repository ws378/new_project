# shelfAwareTurnCost 算法设计

本目录只保存 `shelf_aware_turn_cost` 这一候选增强方案的长期算法真值。这里不是执行记录、迁移计划、实验流水账、弱门禁矩阵或历史失败证据仓库。

`shelf_aware_turn_cost` 的定位是：在 `shelf_aware` 可用基线之上，把 turn-cost 研究中已经进入当前候选设计或被验证为值得保留的方向融入覆盖路径主流程，包括规则覆盖图、转角代价、方向/CTG 评分、来源可追溯和效果守卫。线间距稳定性、lane family 和覆盖贡献建模仍有一部分属于诊断守卫或后续缺口，不能被理解为已经完整进入当前代码模型。它不是论文 official 流程的直接接入，也不是研究脚本后处理结果的 UI 包装。

## 维护原则

- 主题文档描述“应长期成立的设计”，不记录“某一批怎么改、哪个脚本怎么跑、某次审查怎么说”。
- 历史实验只在它能支撑当前设计取舍时被吸收为结论；原始流水账不进入本目录。
- 旧实现边界只在它仍是必要运行时边界时记录；非必要向后兼容不作为保留理由。
- 任何效果相关的设计变更必须回到固定基线、覆盖率、线距、转角、长跳跃、几何只读诊断和运行时间上验证。
- connector、fallback、postprocess 不是额外主算法分支；它们只能作为路径实现中的来源、风险或受限阶段语义。
