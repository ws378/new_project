# 路径实现与诊断产物

## 1. FinalPathRealization 定位

`FinalPathRealization` 的职责是把遍历序列转成正式路径点，并记录必要 provenance。它不是研究后处理平台。

允许的行为：

- 坐标系还原；
- 起止航段对齐；
- 去除确定无意义的重复点；
- 按工程接口输出路径点；
- 记录每段路径来源和必要 transform。

禁止的行为：

- 全路径吸附；
- 无约束删点；
- 批量 shortcut；
- batch reconnect；
- lane 平移后处理；
- 单 case 局部重连；
- 未经过验收的几何风险修复。

## 2. path segment provenance

每个最终路径段都应尽量关联来源。推荐层级：

| 层级 | 说明 |
| --- | --- |
| cell provenance | 该路径点或路径段来自哪些 coverage cell。 |
| move provenance | 该段来自哪一次 traversal move。 |
| source label | coverage、connector-like、fallback-like、final-transform 等来源标签。 |
| risk label | 急转、长跳跃、线距异常、几何只读风险等风险标签。 |
| transform record | 最终路径实现阶段是否移动、合并或删除过点。 |

provenance 的目标是解释，不是给补丁开口。来源标签越多，不代表算法越强，反而可能说明主流程不自然。

## 3. diagnostics summary

diagnostics summary 应给 UI、批量实验和人工审查提供统一摘要。长期字段应覆盖：

- mode / profile；
- 输入地图、区域和参数快照；
- 覆盖图规模；
- lane spacing 统计；
- traversal 候选和异常转移统计；
- path transform 统计；
- 覆盖率、路径长度、总转角、长跳跃；
- 几何只读风险摘要；
- artifact manifest 路径。

如果 diagnostics 中的数字无法追溯到某个阶段或某个路径段，该字段不应进入长期契约。

## 4. artifact 分类

artifact 只做证据展示和回归辅助。推荐分类：

| 类别 | 内容 |
| --- | --- |
| pipeline trace | 阶段输入输出摘要、失败和降级。 |
| graph debug | 覆盖单元、lane family、方向场。 |
| traversal debug | 候选评分、move 来源、访问顺序。 |
| final path debug | 最终路径、provenance、transform。 |
| quality debug | 线距、转角、长跳跃、交叉和几何只读风险。 |
| manifest | 所有 artifact 的 schema、版本、参数和来源。 |

artifact 不直接提交大体积 output。需要版本管理的只应是轻量夹具、schema 示例或文档图。

## 5. schema version

schema 版本用于保护诊断和批量分析脚本，不用于给旧算法行为兜底。

变更规则：

- 字段新增且旧消费者可忽略：小版本变更；
- 字段语义变化或删除：大版本变更；
- 同一字段不能在不同脚本中表达不同语义；
- manifest 必须记录 schema version 和生成脚本。

## 6. 只读几何诊断关系

机器人几何与真实清扫覆盖诊断是只读评价层。它可以暴露：

- 车体扫掠碰撞；
- 转弯 yaw 扫掠风险；
- 盘刷覆盖；
- 刮水器覆盖；
- buffer 覆盖与真实清扫覆盖差异。

它不能在未完成规划语义设计前直接反向修改路径。若未来要把几何约束迁入 planner，必须先形成新的输入契约、状态模型和验收标准。

## 7. UI/export 展示

UI/export 应展示能帮助用户判断结果是否可信的信息：

- 当前算法；
- profile；
- 关键参数；
- 覆盖率；
- 路径长度；
- 转角摘要；
- 长跳跃；
- 线距风险；
- 几何只读风险；
- artifact 目录。

不应把研究内部图层、临时脚本名、过期 run 目录或历史批次编号直接暴露为产品概念。
