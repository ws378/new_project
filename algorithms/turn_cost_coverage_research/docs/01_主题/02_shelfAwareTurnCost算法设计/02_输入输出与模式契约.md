# 输入输出与模式契约

## 1. planner mode 契约

`shelf_aware_turn_cost` 是独立 planner mode。它不能伪装成 `shelf_aware`，也不能让 `shelf_aware` 在内部隐式启用 turn-cost 研究逻辑。

正式 factory mode 和 UI/router/adapter mode 必须分层表达：

| 层级 | mode | 语义 |
| --- | --- | --- |
| formal factory | `basic` | 基础算法。 |
| formal factory | `shelf_aware` | 保守基线。 |
| formal factory | `shelf_aware_turn_cost` | 候选增强基线。 |
| UI / router / adapter | `channel_topology_graph` | 通道拓扑适配入口，不是 formal factory 直接入口。 |

UI 显示名称可以使用中文或产品化命名，但配置层必须保留稳定 mode 字段，避免导出工程、批量脚本和测试夹具无法对齐。

## 2. profile 契约

`shelf_aware_turn_cost` 的 profile 代表一组经过固定测试集验证的策略组合，不是零散参数默认值集合。

当前候选 profile 的关键事实：

| 字段 | 语义 |
| --- | --- |
| `shelf_node_generation_mode=turn_cost_repaired_grid` | 使用 turn-cost 研究确认有效的规则覆盖节点生成方式。 |
| `shelf_repaired_grid_max_offset_factor=0.28` | 控制局部修复网格偏移幅度，避免为线距修复引入过大几何漂移。 |
| `shelf_ctg_auxiliary_enable=true` | 允许使用 CTG territory / junction 辅助信息，但不能让 CTG 成为不可解释兜底。 |
| `shelf_isolated_jump_cleanup_enable=false` | 关闭孤立跳变点治理，避免和 turn-cost 节点生成、线距均衡目标相互抵消。 |

profile 修改必须被视为算法行为变更，需要重新跑固定基线，不得只改 UI 默认值。

## 3. 输入契约

输入必须来自当前工程正式地图项目、区域选择和已有预处理链路。算法入口需要明确接收：

- 地图分辨率；
- 当前区域 mask / polygon；
- 障碍物膨胀后的可通行结果；
- 覆盖宽度；
- 机器人宽度；
- 当前 UI 保存的 shelf-aware 参数；
- 可选方向场、CTG territory / junction 辅助结果；
- 起点信息，未设置时使用区域中心或正式入口既有规则。

输入契约不允许：

- 在 `shelf_aware_turn_cost` 内部重新做独立地图预处理；
- 私自引入和当前预处理不同的软代价语义；
- 为某个地图区域添加硬编码参数；
- 把失败的 CTG 辅助静默解释成另一套算法模式。

## 4. 输出契约

主输出必须是工程现有的 `CoveragePlanningResult` 或等价正式结果对象。它至少包含：

- 路径点序列；
- 覆盖统计；
- 失败或降级信息；
- 可供 UI/export 使用的摘要字段。

所有研究产物和诊断产物只能从主输出和中间阶段记录派生，不得反向修改主输出。

## 5. diagnostics 契约

diagnostics 的职责是解释路径怎么来的、哪里有风险、为什么某些阶段降级。它不是路径真值。

推荐长期字段分层：

| 层级 | 内容 |
| --- | --- |
| mode/profile | planner mode、profile id、关键参数快照。 |
| pipeline | 阶段成功/失败、降级原因、耗时。 |
| graph | 覆盖图规模、方向分布、线距风险统计。 |
| traversal | 候选评分摘要、转角代价、异常转移来源。 |
| final_path | transform 记录、路径段 provenance、长跳跃和急转窗口。 |
| geometry_readonly | 车体扫掠、真实清扫覆盖、buffer 覆盖差异。 |

diagnostics 可以用于验收、回归和人工分析，但不能绕过正式 planner 输出。

## 6. UI/export 契约

UI 需要能明确区分 `shelf_aware` 和 `shelf_aware_turn_cost`。用户选择增强方案时，应能在导出结果中看到：

- 当前 mode；
- profile id 或版本；
- 关键参数；
- 是否启用 CTG auxiliary；
- 节点生成策略；
- 孤立跳变治理是否关闭；
- 覆盖率、线距、转角、长跳跃和几何只读摘要。

UI 不应展示研究脚本内部名，也不应把临时诊断图当成正式路径编辑入口。

## 7. router / adapter 边界

adapter 只能做数据结构转换和显式契约校验。它不应包含：

- 算法策略选择；
- 地图语义修复；
- 单 case 特判；
- 失败后静默切换到另一套路径生成逻辑。

当输入缺失或坐标系不一致时，adapter 应返回明确失败或明确降级原因。
