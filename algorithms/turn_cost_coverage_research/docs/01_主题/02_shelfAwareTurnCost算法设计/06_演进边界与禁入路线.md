# 演进边界与禁入路线

## 1. 可以继续演进的方向

后续优化应围绕主流程能力增强，而不是最终路径补丁扩张。

允许继续演进：

- 覆盖图静态模型；
- lane family 和局部线距建模；
- 候选评分归一化；
- 转角代价和方向稳定性；
- traversal provenance；
- artifact schema 和 manifest；
- 几何只读诊断可读性；
- 固定测试集和效果守卫。

这些方向都必须能解释为输入、覆盖图、遍历决策、路径实现或诊断契约的一部分。

## 2. 禁入路线

以下路线不得迁入正式 UI 链路：

- 为某个区域写硬编码修复；
- 只按最终截图调参数；
- 全路径点吸附到另一套节点；
- 过密 lane 直接删除且无覆盖守卫；
- 大范围 shortcut 或 reconnect；
- 把 fallback 设计成独立主算法；
- 把 connector 设计成无限扩展的补丁集合；
- 用 geometry diagnostic 的结果直接改路径；
- 保留旧接口只是为了“不改测试”；
- 让文档中的历史过程继续约束当前实现。

## 3. 必要旧边界判据

旧边界只有同时满足以下条件，才允许暂时保留：

1. 当前正式输出仍依赖它；
2. 删除它会造成可复现行为退化；
3. 它的职责被限制在 adapter、mirror 或外部接口；
4. 它不新增算法决策；
5. 有明确后续替代路径或停止理由；
6. 验收文档中能说明保留风险。

不满足这些条件的旧边界应删除或吸收到新结构中。

## 4. 行为变更门槛

任何影响路径形态的变更都必须至少提供：

- 固定用例结果；
- 与 `shelf_aware` 和当前 `shelf_aware_turn_cost` 基线对比；
- 覆盖率、线距、转角、长跳跃、几何只读风险；
- 失败或退化样例说明；
- 是否允许进入 UI 默认配置的结论。

只改代码结构但无法证明路径效果不退化，不能算完成。

## 5. 停止条件

某一轮算法内核重构可以停止，必须同时满足：

- 主流程阶段边界清晰；
- 静态覆盖图和动态遍历状态没有双重真值；
- 旧 `Node` 只作为必要 legacy shell 或 semantic 过渡边界；
- provenance 能解释关键路径段；
- 验收指标不低于当前候选基线；
- 代码事实、主题设计、验证文档一致；
- 没有新增非必要向后兼容。

若仍存在效果退化或核心模型不清，不能用“文档已经整理”作为收敛理由。

## 6. 与缺口文档的关系

当前仍未解决的问题应进入 `docs/03_缺口/`，不应继续塞进本目录扩成新设计长文。

本目录只保留已经形成长期设计判断的内容；开放问题必须有状态、影响范围和关闭验证方式。
