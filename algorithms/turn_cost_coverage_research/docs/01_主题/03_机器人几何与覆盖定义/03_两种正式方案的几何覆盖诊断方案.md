# 两种正式方案的几何覆盖诊断方案

## 1. 目标

在不修改规划结果的前提下，对 `shelfAware` 和 `ShelfAware+TurnCost` 增加同一套只读诊断，判断规划路径在真实机器人几何下的风险。

重点回答：

- 车体刚性 footprint 沿路径平移采样时是否会扫到障碍；
- 转弯、掉头、急转窗口中，车体 yaw 插值扫掠是否会扫到障碍；
- 车体 footprint 是否存在紧贴障碍风险；
- 路径中心线 buffer 覆盖率和真实清扫 footprint 覆盖率差多少；
- 盘刷和刮水器分别贡献了多少覆盖；
- `ShelfAware+TurnCost` 的规则节点是否在真实 footprint 诊断下仍优于 `shelfAware`。

诊断判定层不依赖路径语义归因。本文只判断“是否碰撞、是否紧贴、是否漏扫、是否存在急转/连续折线/方向突变”，不判断这些问题属于哪类路径语义。

本文的窗口和 swept footprint 规则不是新的规划算法。它们只把当前工程已经存在的转角、短锯齿、长跳、路径姿态和 footprint 检查思想整理成可重复的只读诊断口径。外部导航框架和覆盖路径文献只作为原则依据，不直接决定阈值；阈值必须优先由本工程地图分辨率、路径点间距、覆盖宽度、机器人宽度、既有障碍膨胀结果和现有 turn constraint 参数推导。

## 2. 非目标

本文不做：

- 不改 `shelfAware` 路径生成；
- 不改 `ShelfAware+TurnCost` 节点生成；
- 不把诊断结果回写路径；
- 不把刮水器纳入硬碰撞；
- 不做单区域专用补丁；
- 不把 `ShelfAware+TurnCost` 加入 `auto`；
- 不实现基于场景语义的局部优化。

## 3. 输入

### 3.1 路径输入

每个 case 至少需要：

- `path_pixels.json` 或正式 planner 输出的 path pixels；
- 地图分辨率；
- 地图 origin；
- 目标区域 mask；
- 预处理后的可通行图；
- 路径点 yaw。

如果路径没有可靠 yaw，应先按相邻路径点估算局部方向，并在诊断元数据中标注 `yaw_source=estimated_from_path`。

### 3.2 机器人几何输入

来自 `01_主题/03_机器人几何与覆盖定义/02_机器人几何与覆盖定义输入契约.md`：

- 后轴中心参考点；
- 车体前后左右边界；
- 障碍膨胀结果和紧贴风险诊断阈值；
- 左右盘刷圆心和半径；
- 刮水器软覆盖多段线或弧线；
- 刮水器是否参与覆盖；
- 刮水器是否参与碰撞；当前只读诊断约定为 `false`。

### 3.3 mask 口径

诊断输出必须记录：

- `collision_mask_policy`：车体碰撞和紧贴检查使用 `free_mask`、`prepared_map` 还是其他障碍口径。
- `target_mask_policy`：清扫覆盖目标区域口径。推荐 `region_mask AND prepared_map`。
- `robot_width_m`：只用于说明既有通行预处理和障碍膨胀口径，不参与真实清扫覆盖率计算。
- `coverage_width_m`：只用于生成中心线 buffer 对照指标，不替代盘刷和刮水器真实清扫 footprint。

## 4. 诊断步骤

### 4.1 路径姿态采样

对路径做等距采样：

- 默认采样间距不大于地图分辨率；如果性能不足，最多放宽到地图分辨率的 2 倍，并在 `pose_sampling_meta.json` 记录实际采样间距；
- 每个采样点保留位置、yaw 和原始路径 index；
- 对长段保留等距采样；
- 对急转窗口、掉头窗口和短距离连续折线窗口增加局部密采样。

输出：

- `pose_samples.json`；
- `pose_sampling_meta.json`。

### 4.2 通用转角窗口检测

不能只看 3 点夹角。应同时检测：

- 3 点窗口：识别单点急转；
- 5 点窗口：识别短距离连续折线；
- 7 点窗口：识别局部方向突然变化、小回折和局部扭动。

基础计算定义：

- 相邻段方向 `heading(a,b)=atan2(b.y-a.y, b.x-a.x)`，角度统一归一到 `[-180, 180]`。
- 单点转角 `turn_delta(i)=normalize(heading(p[i],p[i+1])-heading(p[i-1],p[i]))`。
- 单点转角大小 `abs(turn_delta(i))`。
- 窗口路径长度为窗口内相邻点距离之和，单位由 `resolution_m` 转为米。

3 点窗口 `p[i-1], p[i], p[i+1]`：

- `turn_angle_deg = abs(turn_delta(i))`；
- 用于识别单点急转；
- 适合发现突然折一下的局部点，但不能单独判断连续扭动。

5 点窗口 `p[i-2] ... p[i+2]`：

- 计算 `i-1`、`i`、`i+1` 三个局部转角；
- `turn_angle_deg` 取三个局部转角的最大绝对值；
- `turn_angle_sum_deg` 取三个局部转角绝对值之和；
- 若窗口长度较短、局部转角符号存在交替、`turn_angle_sum_deg` 明显大于 `direction_change_deg`，则标记 `is_short_zigzag=true`；
- 用于识别短距离连续折线、小折返和局部锯齿。

7 点窗口 `p[i-3] ... p[i+3]`：

- 比较窗口前半段主方向和后半段主方向；
- `direction_change_deg` 使用入口方向与出口方向的夹角；
- 同时记录窗口内 `turn_angle_sum_deg` 和最大单点转角；
- 用于识别单个点不一定特别急，但连续多个点造成局部方向突然变化、小回折或扭动的情况。

窗口检测必须做重叠合并：

- 同一类风险的相邻窗口如果路径 index 区间重叠或距离很近，应合并成一个风险窗口；
- 主指标统计合并后的窗口数，不统计滑窗原始命中次数；
- JSON 中可保留原始命中数量用于调试，但不能作为对外主指标。

每个转角窗口必须计算可回指路径点、可解释转角来源、可触发 yaw 插值扫掠检查的明细字段。字段 schema 统一见第 5 节，本文其他小节不重复定义字段含义。

`straight_length_before_m` 和 `straight_length_after_m` 的计算口径：

- 从窗口中心向前或向后沿路径累计；
- 只要连续段方向相对当前主方向变化小于阈值，就继续累计；
- 遇到方向突变、长跳、零长度段或路径端点时停止；
- 阈值当前使用工程 `turn_constraint_near_max_turn_deg`；若 case 元数据缺少该参数，则记录为 `turn_threshold_source=missing` 并使用当前默认配置值。

`requires_turn_swept_check=true` 的触发条件：

- 命中急转窗口；
- 命中短距离连续折线；
- 命中 7 点局部方向突变；
- 窗口前后近似直线段过短；
- 窗口附近存在长跳或可疑连接。

该字段只表示需要做更严格的 yaw 插值 swept footprint 诊断，不表示该窗口一定不可执行。

这些字段的组合解释、示意图和真实路径窗口案例见 `01_主题/03_机器人几何与覆盖定义/04_转角窗口字段计算说明.md`。

该步骤对应的急转、短距离连续折线和方向突变主指标见第 5 节。

### 4.3 车体平移 swept footprint 碰撞诊断

对每个路径采样姿态，把车体矩形变换到地图坐标：

- 检查矩形内是否与障碍相交；
- 检查矩形是否进入既有预处理/障碍膨胀后的不可通行区域；
- 使用 `clearance_warning_m` 记录紧贴障碍风险；该阈值只用于诊断提示，不替代规划中的障碍膨胀参数；
- 生成车体平移扫掠 mask；
- 记录发生风险的路径 index、采样姿态和最小 clearance。

实现约束：

- 采样步长必须由地图分辨率和车体 footprint 尺寸决定，当前默认不大于地图分辨率；如果性能不足，最多放宽到地图分辨率的 2 倍，并记录实际采样步长；
- 采样姿态使用路径局部方向或 planner 输出 yaw，来源必须写入元数据；
- 碰撞 mask 使用 3.3 中记录的 `collision_mask_policy`，不能临时切换；
- `clearance_warning_m` 只用于紧贴诊断提示，不参与路径规划，不替代既有障碍膨胀；
- 平移采样事件、合并风险窗口和调试采样点必须使用不同字段名，避免把事件密度、风险区域数量和采样密度混淆。

可靠性边界：

- 该步骤能发现沿折线路径平移时车体 footprint 与障碍的相交或紧贴；
- 该步骤不能覆盖转弯过程中 yaw 连续变化造成的外扫，因此必须与 4.4 配合；
- 如果路径 yaw 是从相邻点估算的，诊断是对该估算姿态的风险评估，不是控制器真实姿态证明。

该步骤对应的平移采样事件主指标见第 5 节。如果未来要输出合并后的平移风险窗口，应新增独立字段，不能改变现有采样事件字段语义。

### 4.4 转弯/掉头 yaw 插值 swept footprint 诊断

仅沿折线路径方向采样不足以覆盖转弯过程。对 4.2 中标记为 `requires_turn_swept_check` 的窗口，应额外做 yaw 插值：

- 取窗口前入射方向和窗口后出射方向；
- 在同一参考点附近对 yaw 做插值；
- 对每个插值 yaw 放置车体矩形；
- 检查插值过程中的车体 footprint 是否碰撞或紧贴；
- 记录最大转角、插值 yaw 数量、最小 clearance 和对应路径 index。

yaw 插值口径：

- 入口 yaw 取窗口进入方向，出口 yaw 取窗口离开方向；
- 插值方向使用最短角度差；
- 插值参考点默认取窗口中心路径点；如果后续控制器明确使用圆弧或其他运动模型，应单独记录运动模型并重新定义插值参考；
- yaw 步长由地图分辨率和 footprint 外接半径推导：

```text
delta_yaw_rad <= resolution_m / max_footprint_radius_m
```

含义是每次 yaw 变化导致车体最远点移动不超过一个栅格尺度，避免角度采样过粗导致漏诊断。

避免漏诊断的规则：

- 4.2 中所有急转、短锯齿、局部方向突变、短直线接急转、长跳附近转向都必须进入 yaw 插值检查；
- yaw 插值采样必须覆盖入口 yaw、出口 yaw 和中间 yaw；
- 发生风险的相邻 yaw 样本应合并为同一个风险窗口。

避免误诊断的规则：

- 使用与平移 swept footprint 相同的碰撞 mask 口径；
- 紧贴风险和硬碰撞分开统计；
- 不把刮水器纳入硬碰撞；
- 不把该诊断解释为控制器真实轨迹一定碰撞，只解释为“按当前折线转向假设存在车体外扫风险”；
- 如果控制器会做平滑、限曲率、倒车或原地旋转，必须在元数据中记录 `motion_model_assumption`，并在下游仿真或实车链路复核。

该步骤对应的转弯/掉头 yaw 插值扫掠主指标见第 5 节。

### 4.5 清扫 footprint 覆盖诊断

对每个采样姿态，把左右盘刷圆和刮水器软覆盖变换到地图坐标：

- 累积盘刷覆盖 mask；
- 累积刮水器覆盖 mask；
- 合并得到真实清扫 footprint 覆盖 mask；
- 与目标区域 mask 对比。

目标区域应使用 `region_mask AND prepared_map`，不能把障碍内部计入漏扫。

该步骤对应的清扫 footprint 覆盖、漏扫和中心线 buffer 对比主指标见第 5 节。

### 4.6 风险窗口明细边界

风险窗口明细必须避免只输出全局计数。每个窗口至少要能解释转角来源、局部路径长度、转弯前后直线段、长跳干扰、车体最小 clearance、平移扫掠风险、yaw 插值扫掠风险和清扫漏扫。字段 schema 统一见第 5 节。

`near_narrow_passage` 和 `near_long_jump` 是几何/路径序列特征，不要求依赖 lane endpoint、junction、connector 或 fallback 语义。

不进入当前必要字段的内容：

- `lane_endpoint`、`junction`、`connector`、`fallback` 等语义归因；
- 外部控制器内部轨迹代价；
- 基于单个地图区域人工框选的专用标签。

这些内容可以作为后续归因层或优化层输入，但不能混入基础几何诊断判定，避免不同场景使用不同判定标准。

## 5. 输出字段 schema

本节是本文唯一的输出字段 schema。第 4 节只描述计算步骤和产物类别，不重复维护字段含义。

### 5.1 `summary.json` 主指标

当前主指标应保持场景无关：

- `body_swept_collision_count`：车体刚性矩形 footprint 沿路径平移采样时，与不可通行区域发生硬碰撞的采样事件数。当前不是合并窗口数。
- `body_tight_clearance_count`：车体 footprint 未硬碰撞，但与障碍距离低于 `clearance_warning_m` 的平移采样事件数。当前不是合并窗口数。
- `turn_swept_collision_count`：转弯、掉头、急转窗口内，车体 yaw 插值扫掠与不可通行区域发生硬碰撞的窗口数量。
- `turn_swept_tight_clearance_count`：转弯、掉头、急转窗口内，车体 yaw 插值扫掠未硬碰撞但距离障碍低于 `clearance_warning_m` 的窗口数量。
- `sharp_turn_window_count`：3 点窗口识别出的单点急转数量。
- `continuous_zigzag_count`：5 点窗口识别出的短距离连续折线或小回折数量。
- `direction_change_window_count`：7 点窗口识别出的局部方向突变数量。
- `cleaning_footprint_coverage_ratio`：盘刷与刮水器真实清扫 footprint 对目标可清扫区域的覆盖比例。
- `cleaning_footprint_gap_area_m2`：真实清扫 footprint 未覆盖到的目标可清扫区域面积，单位平方米。
- `brush_coverage_ratio`：仅使用左右盘刷圆 footprint 扫掠得到的目标可清扫区域覆盖比例。
- `squeegee_coverage_ratio`：仅使用刮水器软覆盖 footprint 扫掠得到的目标可清扫区域覆盖比例。
- `buffer_coverage_ratio`：使用路径中心线按 `coverage_width_m` 生成 buffer 后得到的目标可清扫区域覆盖比例。
- `buffer_coverage_vs_cleaning_coverage_delta`：`buffer_coverage_ratio - cleaning_footprint_coverage_ratio`，用于判断中心线 buffer 是否高估或低估真实清扫覆盖。

路径语义归因指标不进入当前主指标。解释问题来源时，应作为独立归因文档或独立诊断层设计，不混入本文的基础判定指标。

### 5.2 风险窗口明细字段

风险窗口 JSON 至少包含：

- `window_start_index`：窗口在原始路径中的起点 index。
- `window_end_index`：窗口在原始路径中的终点 index。
- `center_index`：窗口中心路径点 index。
- `window_size`：窗口大小，取值为 `3`、`5` 或 `7`。
- `window_length_m`：窗口内折线路径长度。
- `entry_heading_deg`：窗口入口方向。
- `exit_heading_deg`：窗口出口方向。
- `turn_angle_deg`：窗口内最大单点转角。
- `turn_angle_sum_deg`：窗口内局部转角绝对值之和，用于识别连续折线和小回折。
- `direction_change_deg`：窗口入口方向和出口方向变化。
- `straight_length_before_m`：转弯前可用近似直线段长度。
- `straight_length_after_m`：转弯后可用近似直线段长度。
- `max_segment_length_m`：窗口内最大相邻点距离，用于识别长跳干扰。
- `is_short_zigzag`：是否为短距离连续折线。
- `risk_reason`：风险原因集合。
- `requires_turn_swept_check`：是否需要进入 yaw 插值扫掠检查。
- `min_body_clearance_m`：车体 footprint 与障碍之间的最小距离。
- `body_collision`：车体平移扫掠是否硬碰撞。
- `turn_swept_collision`：yaw 插值扫掠是否硬碰撞。
- `brush_gap_area_m2`：盘刷未覆盖面积。
- `squeegee_gap_area_m2`：刮水器未覆盖面积。
- `cleaning_gap_area_m2`：真实清扫 footprint 未覆盖面积。
- `near_narrow_passage`：转弯窗口是否发生在局部窄通道内。
- `near_long_jump`：转弯前后是否存在长跳跃。

## 6. 可视化

每个 case 输出至少六张图：

- `body_swept_collision_overlay.png`：车体平移扫掠碰撞和紧贴风险。
- `turn_swept_collision_overlay.png`：转弯/掉头 yaw 插值扫掠风险。
- `brush_coverage_overlay.png`：盘刷 footprint 覆盖区域。
- `squeegee_coverage_overlay.png`：刮水器 footprint 覆盖区域。
- `cleaning_gap_overlay.png`：真实清扫漏扫区域。
- `buffer_vs_cleaning_delta_overlay.png`：路径 buffer 覆盖与真实清扫覆盖差异。

可视化必须保证路径本身清晰可读：

- 原路径：高对比蓝色实线，不添加描边。
- 普通车体采样 footprint：青色实线细框。
- `hard_collision`：正红色实线矩形框加红色圆点。
- `tight_clearance`：洋红色虚线矩形框。
- 清扫漏扫：黄色实心栅格区域，单独成图。
- 盘刷覆盖：绿色实心栅格区域，单独成图。
- 刮水器覆盖：蓝色实心栅格区域，单独成图。
- 长跳跃或可疑连接：红色虚线。
- 所有诊断符号、路径线和栅格覆盖层均使用 `opacity=1`。

禁止：

- 用大面积粗红色块遮挡路径；
- 让路径线与风险颜色相同；
- 使用透明度区分关键信息；
- 给路径添加描边；
- 把盘刷覆盖、刮水器覆盖和漏扫区域叠在同一张栅格图里；
- 在图例里写了某类符号但图中没有真实绘制；
- 把所有信息堆在一张总览图里导致不可读。

## 7. 对比口径

同一地图区域必须同时跑：

- `shelfAware`；
- `ShelfAware+TurnCost`。

对比时按以下优先级判断：

1. 车体硬碰撞数量不能增加。
2. 转弯/掉头扫掠硬碰撞数量不能增加。
3. 车体紧贴风险不能显著增加。
4. 真实清扫 footprint 覆盖率不能相对中心线 buffer 覆盖率明显劣化。
5. 漏扫面积不能集中在窄通道、边界或路口关键区域。
6. 若覆盖率和安全性相近，优先选择线距更均匀、急转窗口更少的方案。

## 8. 验证关系

本文只维护几何诊断的方法、字段真值和可视化底线，不维护当前有效用例集合、样例输出路径、批量证据或验收状态。几何诊断验证入口、用例选择、可读性样例、批量结果和当前实现限制由 `05_验证/05_只读几何诊断验证记录.md` 维护。

诊断输出的最低要求是：能回指路径点 index，能区分平移 swept footprint 风险与转弯/掉头 yaw 插值 swept footprint 风险，能区分车体硬碰撞、紧贴风险和清扫漏扫，并记录参数来源、单位、坐标系、mask 口径和 yaw 来源。
