# 开发治理说明

`04_开发` 只保存长期仍需要追溯的治理决策。这里不保存当前推进恢复卡、技能草案、文档治理流水账或历史批次记录。

## 职责

- 记录为什么采用当前文档组织方式；
- 记录为什么 `shelf_aware_turn_cost` 只能作为 candidate enhancement；
- 记录为什么算法重构必须以基线和回归守卫为准；
- 指向长期事实源，不复制执行过程。

## 文件组织

| 目录 | 职责 |
| --- | --- |
| `决策记录/` | 长期治理决策，记录为什么这样组织，不记录单次命令输出。 |

## 禁止内容

- 不把开发计划写成算法设计；
- 不把执行流水长期保留；
- 不新增“当前推进”“历史批次”“技能草案”目录保存已吸收过程；
- 不新增平行的总入口。
