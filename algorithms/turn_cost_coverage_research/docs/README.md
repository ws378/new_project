# turn cost 覆盖研究 docs 说明

## 1. 文档系统定位

`docs` 是 `turn_cost_coverage_research` 当前唯一长期文档入口。

本目录不再按早期复现、研究治理、重构治理、执行记录等历史过程分层维护，而是按长期问题域组织：

- 当前算法入口和能力边界；
- `shelf_aware_turn_cost` 算法设计；
- 机器人几何与清扫覆盖定义；
- 当前正式代码入口、研究代码生命周期、output 证据资产规则和验证事实；
- 当前缺口、历史拒绝路线和治理决策；
- 论文、第三方来源和人工标注参考。

旧文档原文不作为当前真值保留。有效内容必须吸收到 `01_主题/`、`02_代码事实/`、`03_缺口/`、`05_验证/`、`reference/` 或仓库级 `docs/governance/`。

## 2. 核心原则

1. 一个长期问题只有一个主入口。
2. 主题文档按算法认知模型组织，不按旧阶段、旧实验或单次讨论组织。
3. 当前事实必须能回链到源码、脚本、测试、配置、output manifest、baseline 或 reference。
4. 旧文档中的有效设计、失败经验、验证口径和治理规则必须被吸收；已吸收原文不长期保留。
5. 当前推进和治理材料不伪装成算法真值；稳定结论必须进入主题、代码事实、缺口或验证文档。
6. reference、机器人几何、当前基线、算法设计主体是长期资产，不能因为清理比例被误删。
7. 新增文档、目录和标题默认使用中文；技术标识保持原文。

## 3. 一级目录职责

| 目录 | 职责 | 不负责 |
| --- | --- | --- |
| `01_主题/` | 主阅读链的主题正文区，承载算法入口、算法设计、机器人几何与覆盖定义。 | 不放执行日志、旧实验长文、当前推进计划。 |
| `02_代码事实/` | 当前正式代码入口、研究代码生命周期和 output 证据资产规则。 | 不写长期方案，不替代算法设计，不保存一次性扫描摘要。 |
| `03_缺口/` | 当前未闭环问题和历史拒绝路线索引。 | 不承载长篇设计论证，不保存旧讨论全文。 |
| `04_开发/` | 长期治理决策记录。 | 不保存当前推进、技能草案、执行流水、历史批次长表。 |
| `05_验证/` | 验收清单、测试分层、GUI 验收边界、当前基线和门禁口径。 | 不保存无索引的单次日志堆积，不反向定义算法主流程。 |
| `reference/` | 论文、翻译、第三方来源、人工标注证据和参考图。 | 不自动成为当前工程方案。 |

## 5. 来源状态

旧文档和历史实验最多使用三类状态：

| 状态 | 含义 |
| --- | --- |
| `待吸收` | 仍有有效内容，尚未完整吸收到当前 docs。 |
| `历史证据` | 只用于追溯当时判断，不提供当前方案判断。 |
| `已关闭` | 有效内容已吸收，或确认没有继续阅读价值。 |

不再增加更细状态，避免阅读和维护成本失控。

## 6. 禁止事项

1. 不恢复早期复现、研究治理、重构治理、工程治理、归档说明、历史批次等旧目录结构。
2. 不把旧文档原文整体搬回 active 文档系统。
3. 不把单次执行记录写入主阅读链。
4. 不把开发计划写成当前代码事实或长期算法设计。
5. 不把失败实验、单 case 结果、临时优化路线提升为当前主线。
6. 不新增没有生命周期、没有唯一职责、不能回链证据的文档。

## 7. 常用术语速查

| 术语 | 中文说明 | 使用边界 |
| --- | --- | --- |
| `shelf_aware` | shelfAware 保守正式基线。 | 不等于早期所谓“原版”长期并存；当前 UI 名称按正式基线理解。 |
| `shelf_aware_turn_cost` | ShelfAware+TurnCost 候选增强入口。 | 不等于论文 official turn-cost planner，也不等于研究脚本最终结果。 |
| `profile` | 算法配置画像。 | 用于说明一组固定策略和参数来源，不允许散落成隐式参数拼接。 |
| `formal planner` | 正式 planner 链路。 | 只能使用经入口契约和验证门禁承认的代码路径。 |
| `research script` | 研究脚本。 | 可用于试验、诊断、复盘；不能自动成为 UI 主线。 |
| `artifact` | 可审计产物。 | 是证据容器，不自动等于当前算法真值。 |
| `baseline` | 基线。 | 用于效果和行为守卫；不能被普通单次 output 替代。 |
| `candidate` | 候选结果。 | 需要通过门禁和人工可读检查后才能作为候选基线。 |
| `diagnostic` | 诊断。 | 用于暴露风险和解释问题，不自动改变 planner 行为。 |
| `reference` | 参考来源。 | 提供论文、第三方、人工标注证据；不自动成为当前工程方案。 |
| `fallback` | 降级或兜底语义。 | 必须可追溯来源和原因，不能扩展成平行主流程。 |
| `connector` | 连接语义。 | 用于解释路径实现中的连接段，不作为独立算法分支。 |
