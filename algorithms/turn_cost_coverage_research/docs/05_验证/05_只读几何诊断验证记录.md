# 只读几何诊断验收口径

## 1. 验证目标

只读几何诊断只做评价，不做路径优化，也不回写任何 planner 结果。

诊断对象是两种正式方案：

- `shelfAware`；
- `ShelfAware+TurnCost`。

输出必须同口径，避免把算法差异和诊断口径差异混在一起。

## 2. 诊断入口

### 2.1 单方案只读诊断

入口：

```bash
python3 algorithms/turn_cost_coverage_research/scripts/diagnostics/run_geometry_coverage_readonly_diagnostic.py \
  --run-dir <planner_run_dir> \
  --prepare-dir <area_dir>/preprocess/prepare_map \
  --case-name <case_name>
```

输出：

- `summary.json`：指标汇总；
- `path_window_diagnostics.json`：3/5/7 点路径窗口诊断明细；
- `pose_sampling_meta.json`：平移采样与 yaw 插值采样口径；
- `00_只读几何诊断总览.png`；
- `01_车体扫掠风险.png`；
- `02_转弯yaw扫掠风险.png`；
- `03_转角窗口诊断.png`；
- `04_盘刷覆盖.png`；
- `05_刮水器覆盖.png`；
- `06_真实清扫漏扫.png`；
- `07_buffer与真实清扫覆盖差异.png`；
- `08_风险窗口放大.png`。

### 2.2 两方案同口径对比

入口：

```bash
python3 algorithms/turn_cost_coverage_research/scripts/diagnostics/run_geometry_coverage_plan_comparison.py \
  --case-name <case_name> \
  --shelf-aware-run-dir <shelfAware_area_dir> \
  --turn-cost-run-dir <ShelfAware+TurnCost_area_dir>
```

该入口会分别调用单方案诊断，并额外输出：

- `<case_name>_两方案几何覆盖对比/comparison_summary.json`。

## 3. 路径窗口字段实现

路径窗口计算下沉到：

```text
algorithms/turn_cost_coverage_research/src/diagnostics/path_window_diagnostics.py
```

当前实现包含：

- 3 点窗口：识别单点急转；
- 5 点窗口：识别短距离连续折线；
- 7 点窗口：识别局部方向突变、小回折和局部扭动；
- 重叠窗口合并：把连续命中的相邻窗口合并为一个风险窗口；
- `requires_turn_swept_check`：用于触发后续 yaw 插值 swept footprint 检查。

字段口径以 `01_主题/03_机器人几何与覆盖定义/04_转角窗口字段计算说明.md` 为准。

## 4. 几何诊断字段关系

`summary.json` 字段真值以 `01_主题/03_机器人几何与覆盖定义/03_两种正式方案的几何覆盖诊断方案.md` 为准。本文不重复维护字段清单，只记录当前实现和验证时必须注意的差异：

- `body_swept_collision_count` 与 `body_tight_clearance_count` 当前是平移采样事件数，不是合并后的风险窗口数。
- `turn_swept_collision_count` 与 `turn_swept_tight_clearance_count` 当前是合并后的转弯风险窗口数。
- `turn_swept_collision_candidate_event_count` 与 `turn_swept_tight_clearance_candidate_event_count` 只用于调试窗口密度，不作为正式验收主指标。
- `collision_mask_policy`、`target_mask_policy`、`pose_sampling_policy` 必须写入输出，用于复核 mask、采样步长、yaw 来源和运动模型假设。

## 6. 用例选择和诊断输出合格条件

几何诊断用例至少覆盖：

- 路口、端部掉头、窄通道和长直通道；
- `shelfAware` 与 `ShelfAware+TurnCost` 同一区域对比；
- 可快速人工检查的可读性样例；
- 三项目 19 区域的批量证据。

只读诊断输出必须满足：

- 每个 case 输出完整 JSON、CSV 和 overlay；
- 风险窗口能回指到路径点 index；
- 指标能区分 `shelfAware` 和 `ShelfAware+TurnCost`；
- 能区分平移 swept footprint 风险和转弯/掉头 yaw 插值 swept footprint 风险；
- 刮水器不参与硬碰撞；
- 盘刷和刮水器覆盖可分别统计；
- 中心线 buffer 覆盖率和真实清扫覆盖率差值可量化；
- 坐标系、单位、mask 口径、yaw 来源和机器人几何参数来源明确。

## 8. 诊断限制

1. 机器人几何参数仍是样例配置，后续进入正式结论前需要替换为标定参数。
2. `body_swept_collision_count` 与 `body_tight_clearance_count` 是平移采样事件数，不是合并后的窗口数；目前用于定位风险密度，不能直接等价为“风险区域数量”。
3. 转弯 yaw 插值采用窗口中心参考点原地插值，用于暴露当前 planner 没有检查的转弯扫掠风险；它不是完整运动学仿真。
4. 清扫 footprint 覆盖仍基于离散采样栅格，采样步长当前固定为 1 px，后续如性能不足可以放宽，但必须记录采样策略。

## 9. 后续验证建议

1. 补充真实机器人几何标定参数，并把参数来源写入输入契约。
2. 对全区域汇总表做排序视图，例如按 `turn_swept_collision_count`、`cleaning_footprint_gap_area_m2` 和 `sharp_turn_window_count` 找优先复核区域。
3. 对风险图中高频区域做人工抽样复核，区分“地图预处理边界噪声”“路径形态风险”和“真实几何风险”。
4. 只在诊断口径稳定后，再讨论是否把几何风险作为 UI 提示或 planner 后置守卫。
