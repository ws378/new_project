# 研究代码与脚本生命周期

本文定义 `turn_cost_coverage_research/scripts/` 和研究 `src/` 的生命周期规则。它不逐项罗列所有脚本；脚本实际列表以文件系统和测试为准。

## 1. 生命周期类别

| 类别 | 目录 | 当前语义 | 是否可进入 UI 主线 |
| --- | --- | --- | --- |
| `baseline` | `scripts/baseline/` | 生成基线、候选门禁和交付门禁证据。 | 只能通过 formal mode 间接体现。 |
| `diagnostics` | `scripts/diagnostics/`、`src/diagnostics/` | 只读诊断、可视化、人工标注对齐、output manifest。 | 否；只输出证据。 |
| `experiments` | `scripts/experiments/`、`src/experiments/` | 仍可复跑的研究实验和候选算子。 | 否；除非重新设计并迁入主工程。 |
| `archived` | `scripts/archived/` | 已否定或不再推荐的历史实验入口。 | 否。 |
| `third_party_reference` | `third_party/`、`docs/reference/` | 论文、官方实现、历史依赖和来源说明。 | 否；只能作为来源证据。 |

## 2. baseline 脚本

baseline 脚本只允许做两件事：

- 生成或检查 `shelf_aware`、`shelf_aware_turn_cost` 的当前基线证据；
- 串联正式入口、契约测试和多区域效果门禁。

当前关键入口：

| 入口 | 中文说明 |
| --- | --- |
| `scripts/baseline/run_shelf_aware_all_areas.py` | 三项目多区域基线生成和对比入口。 |
| `scripts/baseline/run_candidate_baseline_gate.py` | 候选基线快速门禁入口。 |
| `scripts/baseline/run_shelf_aware_turn_cost_delivery_gate.py` | `shelf_aware_turn_cost` 最终交付门禁入口。 |

baseline 脚本不得包含新的路径改写策略。如果需要新增候选算法，应先进入 `experiments`，再经过设计、迁移和验收。

## 3. diagnostics 脚本

diagnostics 脚本必须满足：

- 不改变输入路径；
- 不输出“优化后路径”作为 planner 结果；
- 可以输出图、JSON、summary 和风险窗口；
- 诊断结论只能进入缺口、验证或设计依据，不能直接改变 formal planner 行为。

几何只读诊断、路径质量诊断、人工标注对齐和 output manifest 都属于这一类。

当前需要长期识别的 diagnostics 能力：

完整脚本列表以文件系统为准；下表只列后续维护必须识别的能力入口，避免重新退化为逐项流水账。

| 能力 | 代表入口 | 中文说明 |
| --- | --- | --- |
| output manifest | `scripts/diagnostics/build_output_asset_manifest.py` | 扫描 output 证据资产，生成本地 ignored manifest。 |
| 效果快照 | `scripts/diagnostics/build_effect_snapshot.py` | 对候选结果和基线结果做指标快照。 |
| 指标契约 | `scripts/diagnostics/effect_metric_contract.py` | 统一候选门禁使用的指标字段和比较口径。 |
| 几何只读诊断 | `scripts/diagnostics/run_geometry_coverage_readonly_diagnostic.py` | 输出车体/清扫 footprint 风险证据，不修改路径。 |
| 路径质量诊断 | `scripts/diagnostics/run_shelf_path_turn_cost_diagnostics.py` | 输出路径窗口、转角、线距、来源等诊断证据。 |

## 4. experiments 脚本

experiments 脚本可以探索：

- official 论文流程复现；
- HiGHS/Gurobi 替代实验；
- lane family、局部重连、节点吸附、shortcut、post opt；
- 参数 sweep 和单 case 对比。

但 experiments 的结论必须经过吸收：

```text
实验结果 -> 缺口或设计结论 -> 主工程实现方案 -> 契约测试和多区域门禁
```

禁止把 experiments 结果直接接成 UI 选项或默认行为。

## 5. archived 脚本

archived 只保留还有复盘价值的历史入口。满足下面任一条件时，应删除而不是归档：

- 干货已经被长期文档吸收；
- 没有唯一复现价值；
- 只记录旧参数和旧 run 路径；
- 继续保留会让维护者误以为它是可推荐入口。

archived 脚本不保证长期可执行。若某个 archived 脚本被测试或文档重新依赖，必须先说明为什么它不是误用。

## 6. 研究 `src` 生命周期

研究 `src` 与脚本生命周期一一对应：

| 目录 | 生命周期 | 中文说明 |
| --- | --- | --- |
| `src/adapters/` | `experiments_support` | 支撑 official/MapTools 复现实验的数据适配，不是 formal planner adapter。 |
| `src/diagnostics/` | `diagnostics` | 只读诊断实现，可被 baseline gate 读取指标，但不得改写路径。 |
| `src/experiments/` | `experiments` | 研究性路径改写和候选算子，默认禁止进入 UI 主线。 |
| `src/guidance/` | `experiments` | 方向引导、square8、official flow 相关研究支撑。 |
| `src/official_replacements/` | `third_party_reference` | official 复现的开源 solver 替代实验支撑。 |
| `src/visualization/` | `diagnostics_support` | 研究产物写出和可视化辅助，不定义算法行为。 |

## 7. 根目录脚本规则

`scripts/` 根目录不再保留 `run_*.py` 兼容 wrapper。根目录只允许：

- `README.md`；
- `__init__.py`；
- 环境检查脚本；
- 分层子目录。

新增脚本必须先选择生命周期类别，不允许先放根目录再事后归类。

## 8. 清理和迁移准则

脚本或研究模块清理时按下面顺序处理：

1. 查是否被主工程、测试、baseline、diagnostics 或当前文档引用；
2. 判断是否有唯一干货；
3. 若有干货，吸收到主题、缺口、验证或代码事实；
4. 若只是过程证据，删除；
5. 若仍需复盘且不推荐使用，放入 archived；
6. 更新受影响的长期入口或仓库级治理文档。

历史证据不是保留理由。能被长期文档吸收的内容，必须吸收后删除原流水。
