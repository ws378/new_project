# 正式代码入口与依赖边界

本文记录 `turn_cost_coverage_research` 与正式覆盖规划代码之间的依赖边界。它不是算法设计文档，也不是脚本清单。

## 1. 正式入口

正式 planner 入口以主工程为准，当前关键文件如下：

| 层级 | 代码位置 | 当前事实 |
| --- | --- | --- |
| mode 注册 | `algorithms/coverage_planning/modes.py` | `basic`、`shelf_aware`、`shelf_aware_turn_cost` 是 formal factory mode；`auto` 是 router；`channel_topology_graph` 是 adapter-only mode。 |
| public config | `algorithms/coverage_planning/contracts/config.py` | `CoveragePlannerConfig` 是 UI、adapter、formal planner 共享的公开参数契约；legacy public/private key 在这里拒绝。 |
| mode profile | `algorithms/coverage_planning/profiles.py` | `shelf_aware_turn_cost` 的 profile 固定 `turn_cost_repaired_grid`、`0.28`、CTG auxiliary on、isolated jump cleanup off。 |
| node generation registry | `algorithms/coverage_planning/node_generation_profiles.py` | `shelf_cell_adjusted` 是 `shelf_aware` baseline；`turn_cost_repaired_grid` 是 `shelf_aware_turn_cost` candidate default。 |
| formal factory | `algorithms/coverage_planning/planner_factory.py` | `create_coverage_planner()` 和 `run_formal_planner_request()` 只直接运行 formal factory mode。 |
| router | `algorithms/coverage_planning/routing/coverage_router.py` | `auto` 和 routed mode 的选择逻辑在这里，不等同于 formal factory mode。 |
| routing adapters | `algorithms/coverage_planning/routing/adapters/` | `channel_topology_graph` 等 adapter-only 能力在这里接入，不由 formal factory 直接创建。 |
| shelfAware planner | `algorithms/coverage_planning/planners/shelf_aware_guarded/shelf_aware_planner.py` | `shelf_aware` 和 `shelf_aware_turn_cost` 共享正式 planner 主体，区别来自 profile/default overrides 和节点生成设置。 |
| shelfAware internal stages | `algorithms/coverage_planning/planners/shelf_aware_guarded/` | 覆盖图、节点生成、遍历、评分、最终路径实现和 artifact 写出在这里。 |
| MapTools adapter | `maptools/adapters/coverage_planning_adapter.py` | MapTools 侧稳定桥接，只 re-export formal planning API。 |
| UI 参数对话框 | `maptools/views/coverage_dialog.py` | UI mode 列表、参数显隐、profile override notice 和 dialog config 转换在这里。 |
| UI 调用和诊断展示 | `maptools/views/main_window.py` | `show_coverage_dialog()` 后构造 request；`auto` 走 router，formal mode 走 `run_formal_planner_request()`；最近一次 diagnostics 用于详情展示。 |
| 覆盖仓库导出 | `maptools/utils/coverage_repo_export.py` | 自动生成覆盖路径时复用 formal planner/router，不单独定义算法行为。 |

`turn_cost_coverage_research` 不是 formal planner 包。它只能提供：

- 研究脚本；
- 只读诊断；
- 可提交的轻量 baseline fixture；
- 迁入主工程后的设计依据和验证证据。

## 2. 正式运行链路允许依赖

正式运行链路可以依赖：

- `algorithms/coverage_planning/**` 中已经接入的 planner、profile、节点生成、遍历和 artifact 代码；
- `maptools/adapters/coverage_planning_adapter.py` 暴露的正式规划桥接 API；
- `maptools/views/coverage_dialog.py`、`maptools/views/main_window.py` 和 `maptools/utils/coverage_repo_export.py` 中调用 formal planner 或 router 的运行入口。

验证和交付门禁可以依赖：

- `algorithms/turn_cost_coverage_research/baselines/**` 中用于门禁对比的轻量 fixture；
- `tests/**` 中明确面向正式链路的契约测试；
- `algorithms/turn_cost_coverage_research/scripts/baseline/**` 下的基线和交付门禁脚本。

正式链路不得直接依赖：

- `turn_cost_coverage_research/src/experiments/**`；
- `turn_cost_coverage_research/scripts/experiments/**`；
- `turn_cost_coverage_research/scripts/archived/**`；
- `turn_cost_coverage_research/baselines/**` 作为运行时算法输入；
- `tests/**` 作为运行时算法输入；
- output 中任意单次 run 的私有文件；
- 论文 official 复现脚本或 HiGHS/Gurobi 替代实验结果。

## 3. 研究包内部边界

研究包内部模块按用途分层：

| 分层 | 当前职责 | 进入正式链路条件 |
| --- | --- | --- |
| `src/adapters/` | MapTools 与 official 复现实验之间的数据适配。 | 不能直接进入 UI planner；若迁入必须重写为主工程契约。 |
| `src/diagnostics/` | 只读路径质量、人工标注、几何覆盖诊断。 | 只能作为诊断或门禁证据；不得直接改写路径。 |
| `src/experiments/` | lane family、局部重连、post opt 等研究实现。 | 默认禁止；必须重新设计为 formal 模块并通过全量验收。 |
| `src/guidance/` | 方向引导、square8、official flow 相关研究。 | 默认禁止；不得作为 `shelf_aware_turn_cost` 当前主线。 |
| `src/official_replacements/` | official 论文流程的开源 solver 替代件。 | 仅用于复现研究，不进入正式 planner。 |
| `src/visualization/` | 研究产物写出和可视化辅助。 | 只能输出证据，不定义算法行为。 |

## 4. 禁止双真值

以下情况一律视为代码事实层 P0/P1 问题：

- 同一 planner mode 同时由主工程和研究脚本定义行为；
- `shelf_aware_turn_cost` 的默认参数只写在研究脚本中，没有进入主工程契约；
- 研究 output 中某次效果好的 run 反向定义正式算法；
- 诊断脚本中的评分阈值被误当成 planner 硬约束；
- archived 或 experiments 入口被 UI、factory 或 formal mode import。

## 5. 核验方式

维护本文件时至少核验下面关系。命令只是核验入口，不是本文的第二套事实来源：

```bash
rg -n "shelf_aware_turn_cost|shelf_aware|channel_topology_graph" \
  algorithms/coverage_planning maptools tests

rg -n "turn_cost_coverage_research\\.src\\.experiments|scripts/experiments|scripts/archived" \
  algorithms/coverage_planning maptools tests
```

第一条用于确认正式入口位置；第二条用于确认研究实现没有被正式链路误引入。
