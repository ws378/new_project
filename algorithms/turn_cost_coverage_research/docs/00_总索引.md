# turn cost 覆盖研究文档总索引

## 1. docs 的职责

`turn_cost_coverage_research/docs` 是 `turn_cost_coverage_research` 的长期知识入口。它不承担以下职责：

- 不保存单次执行流水；
- 不保存已经被吸收的旧讨论全文；
- 不把失败实验原文当作长期资产；
- 不把当前推进记录伪装成算法设计；
- 不替代代码、测试和基线产物本身。

它只保留能长期回答下面问题的内容：

- 当前有哪些可用算法入口，各自能力边界是什么；
- `shelf_aware_turn_cost` 的算法思路、流程、数据结构和禁止事项是什么；
- 当前正式代码入口、研究代码生命周期和 output 证据资产如何分层；
- 当前缺口是什么，哪些历史路线已被拒绝；
- 当前治理决策和重构边界如何追溯；
- 当前验证体系、基线和门禁如何执行；
- 论文、第三方来源、人工标注等参考证据在哪里。

## 2. 一级目录职责

| 目录或文件 | 职责 | 不负责 |
| --- | --- | --- |
| `00_总索引.md` | docs 第一入口；说明目录职责、阅读顺序、仲裁规则。 | 不承载算法细节和执行流水。 |
| `01_主题/` | 长期主题真值；包括算法入口、算法设计、机器人几何与覆盖定义。 | 不记录批次计划、执行日志、旧实验过程。 |
| `02_代码事实/` | 当前正式代码入口、研究代码生命周期和 output 证据资产规则。 | 不提出新算法方案，不保存脚本流水账或扫描摘要。 |
| `03_缺口/` | 当前未闭环问题和已拒绝历史路线索引。 | 不保存长讨论和旧计划原文。 |
| `04_开发/` | 长期治理决策记录。 | 不保存当前推进、技能草案、执行流水、历史批次长表。 |
| `05_验证/` | 验证体系、验收清单、测试分层、GUI 验收、当前基线。 | 不替代测试脚本和真实 output。 |
| `reference/` | 论文、翻译、第三方来源说明、人工标注证据和参考图。 | 不自动成为当前工程方案。 |

## 4. 当前算法入口摘要

当前 UI 中只保留两个 ShelfAware 相关入口：

```text
shelf_aware           # shelfAware，保守正式基线
shelf_aware_turn_cost # ShelfAware+TurnCost，显式选择的候选增强入口
```

`shelf_aware_turn_cost` 不是论文 official turn-cost planner，也不是研究脚本最终结果。它只迁入已确认有效的节点生成替换主线：

- 使用 `ShelfAwareCoveragePlanner` 原路径组织、连接和 CTG 辅助；
- 强制 `shelf_node_generation_mode=turn_cost_repaired_grid`；
- 强制 `shelf_ctg_auxiliary_enable=True`；
- 强制 `isolated_jump_cleanup_enable=False`；
- 不迁入 batch reconnect、全路径吸附、删点、lane 平移或其他研究后处理。

详细边界见 `docs/01_主题/01_算法入口与能力边界.md`。

## 6. 仲裁规则

当文档之间出现冲突时，按以下顺序判断：

1. 当前代码、测试、配置和正式门禁结果优先于任何历史文档。
2. `01_主题/` 是长期主题真值，优先于 `04_开发/` 的推进记录。
3. `02_代码事实/` 只记录实现事实，不提出算法目标。
4. `03_缺口/` 只记录未闭环问题和历史拒绝路线，不替代设计文档。
5. `05_验证/` 定义验收口径，不反向定义算法主流程。
6. `reference/` 只提供来源证据，不自动成为当前工程方案。
7. 已删除的早期研究文档和执行记录只可通过 git 历史追溯，不作为当前真值。

## 7. 当前禁止误用

以下内容不得作为 UI 主线或默认正式能力：

- official turn-cost 完整流程；
- square8 official flow；
- 全路径节点吸附；
- batch reconnect 研究输出；
- lane family 简单平移输出；
- area5 单 case 最优结果；
- 几何只读诊断直接作为 planner 硬约束。

## 8. 维护准则

后续维护必须遵守：

1. 不恢复早期复现、研究治理、重构执行记录等旧目录结构。
2. 不把已吸收的历史文档原文搬回 active 文档系统。
3. 不把单次执行记录写入主阅读链。
4. 不把当前推进计划写成长期算法设计。
5. 不新增没有明确生命周期和唯一入口的文档。
6. 不因为“历史证据”长期保留已被吸收的过程材料。
7. 删除任何文档前，必须确认干货已进入主题、代码事实、缺口、验证、reference 或仓库级 `docs/governance/`；不再用长期文档保存删除流水账。
